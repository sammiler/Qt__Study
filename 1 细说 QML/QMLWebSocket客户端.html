WebSocket 是一种网络传输协议，用于在客户端和服务器之间进行双向通信（全双工）。它为 Web 应用程序提供了一个持久化连接，可以实时地在客户端和服务器之间传输数据，而无需在每次请求时建立新的连接。这种实时通信技术的应用场景非常广泛，如社交聊天、在线游戏、股票行情、协同编辑和体育实况等。

要在 QML 中实现 WebSocket 客户端，需要用到 WebSocket 类型，它是基于 C++/Qt 中的 QWebSocket 实现的。所以，只要知道如何使用 QWebSocket，就能快速上手 WebSocket。

数据传输方式

在进行数据传输时，除了上面提到的全双工之外，常见的方式还有以下几种：

  • 全双工：允许数据在两个方向上同时传输，发送方和接收方可以同时发送和接收数据。例如：打电话，说话的同时也能够听到对方的声音。常见的全双工协议有 WebSocket、TCP/IP 等。

  • 半双工：数据可以沿两个方向传输，但同一时刻只允许单方向传输，即：双向交替通信。例如：对讲机，同一时间只允许一方讲话，另一方只能听着。常见的半双工协议有 HTTP。

  • 单工：数据只能单向传输，从一个方向传输到另一个方向，不能反向传输。例如：广播电台，只能由电台传输信号到收音机，而无法由收音机向电台发送信号。

注意： 关于数据传输的方式，可以从多个维度进行归类。若按数据传输的流向和时间关系，可以分为全双工/半双工/单工数据传输；若按数据传输的顺序，可以分为并行传输和串行传输；若按数据传输的同步方式，可分为同步传输和异步传输。

与 HTTP 的区别

与 HTTP 相比，WebSocket 有以下区别：

  1. 连接方式：HTTP 是一种无状态协议，每次请求都需要建立新的连接，并在服务器端返回响应后关闭。而 WebSocket 在 HTTP 的基础上实现了长连接，通过一个初始 HTTP 握手后，连接将保持打开状态，双方可以随时发送数据。
  2. 实时性：WebSocket 提供了双向实时通信，服务器可以主动向客户端推送数据，客户端也可以主动向服务器发送消息。而 HTTP 是单向的请求-响应模式，客户端需要不断轮询或发送请求来获取数据。
  3. 通信效率：WebSocket 减少了连接建立和头部信息的传输，通过持久连接，节省了网络资源和通信延迟，在通信效率上更高一些。
  4. 协议开销：WebSocket 在使用 HTTP 协议握手后，转换为自定义的协议（二进制帧格式）进行数据传输，有效地减少了数据的传输量，协议开销更小。

总的来说，WebSocket 相比于 HTTP，在实时性、双向通信、通信效率等方面具有明显的优势，特别适用于需要实时更新数据和实时通信的应用。

URI 格式

在 WebSocket 规范 RFC 6455 中，定义了两种 URI 协议 - ws 和 wss，分别用于明文和加密连接：

ws-URI = ws://host:port/path?query
wss-URI = wss://host:port/path?query

各字段含义如下：

  • ws 或 wss：协议
  • host：域名（或 IP 地址）
  • port：端口号
  • path：资源路径
  • query：查询字符串（可选）

其中，ws 是 WebSocket 的缩写，wss 是 WebSocket Secure 的缩写。

也就是说，wss 是 ws 基于 TLS/SSL 的安全传输，相当于 https 对于 http 一样，更加安全、可靠。所以，一般建议首选 wss。

调试工具

在调试 WebSocket 时，建议找一些辅助工具，推荐几个比较好用的：

  • Postman
    一款非常流行的 API 开发和测试工具，不仅支持 HTTP/HTTPS，还支持 WebSocket、Socket.io、EventSourcee、MQTT 等协议。
    Postman 分为免费版和付费版。免费版提供了一些基础功能，例如：创建和分享 API、发送和测试 API 等；付费版则包含了更多高级特性，例如：团队协作、自动化测试等。但对于大部分用户来说，免费版已经足够用了。

  • Hoppscotch
    Postman 的最佳替代品，由印度小哥 Liyas Thomas 开发，开源免费、轻量级、快速且美观。Hoppscotch 前身叫 Postwoman，显而易见，就是专怼 Postman 的（让你收费），后来才改为现在的名字。

注意： 以上两个工具，可以同时调试 WebSocket Client 和 WebSocket Server。

如果开发的是 Client，可以连接下面的地址作为 Echo Server：

  • wss://ws.postman-echo.com/raw
  • wss://echo-websocket.hoppscotch.io
  • wss://echo.websocket.events

基本步骤

在使用 WebSocket 时，基本步骤如下：

  1. 导入 QtWebSockets 模块
import QtWebSockets
  1. 创建 WebSocket 对象，并进行配置。

在创建 WebSocket 时，需要通过 url 属性指定将要连接的服务器地址，并监听 textMessageReceived() 和 statusChanged() 信号，以获取服务器端发送的消息并实时监控连接的状态。

WebSocket {
            
    id: socket

    // 指定服务器地址
    url: "wss://ws.postman-echo.com/raw"

    // 收到服务端的消息时触发
    onTextMessageReceived: (message)=> {
            
        console.info("Received message: " + message)
    }

    // 状态发生变化时触发
    onStatusChanged: (status)=> {
            
        console.info("Status changed: " + status)
    }
}
  1. 连接服务器

为了连接服务器，我们需要将 active 属性设置为 true。然后在必要时调用此函数，例如：点击按钮时。

function connectToServer() {
            
    socket.active = true
}
  1. 发送消息

可以使用 sendTextMessage() 方法发送文本信息，也可使用 sendBinaryMessage() 发送二进制信息。但是在发送时，status 需要是打开状态。

function sendTextMessageToServer(message) {
            
    if (socket.status === WebSocket.Open) {
            
        socket.sendTextMessage(message)
        return true
    } else {
            
        return false
    }
}
  1. 关闭连接

如果想断开与服务器之间的连接，需要将 active 属性设置为 false。

function disconnectFromServer() {
            
    socket.active = false
}

使用 WebSocket

我们来编写一个完整的 WebSocket Client，利用它来模拟聊天：

import QtQuick
import QtQuick.Controls
import QtQuick.Layouts
import QtWebSockets

Rectangle {
            
    width: 500; height: 400

    WebSocket {
            
        id: socket

        // 接受服务端的消息，直接显示
        onTextMessageReceived: (message)=> {
            
	        inner.showMessage(message, true)
        }

        // 状态发生变化时（例如：断开连接），进行提示
        onStatusChanged: (status)=> {
            
            inner.showStatusTip(status)
        }

        // 连接至服务器
        function connectToServer(url) {
            
            socket.url = url
            socket.active = true
        }

        // 断开与服务器的连接
        function disconnectFromServer() {
            
            socket.active = false
        }

        // 发送消息至服务器
        function sendTextMessageToServer(message) {
            
            if (socket.status === WebSocket.Open) {
            
                socket.sendTextMessage(message)
                return true
            } else {
            
                return false
            }
        }
    }

    QtObject {
            
        id: inner

        // 发送消息给服务器（发送后，清空消息输出框，并显示已发送的消息）
        function sendMessage() {
            
            let message = textField.text
            textField.clear()
            let isSuccess = socket.sendTextMessageToServer(message)
            if (isSuccess) {
            
                showMessage(message, false)
            } else {
            
                showTip("socket is not open, please connect first")
            }
        }

        // 显示消息（根据颜色区分，isReceived：true 表示发送的消息，false 表示接收的消息）
        function showMessage(message, isReceived) {
            
            let time = Qt.formatDateTime(new Date(), "yyyy-MM-dd hh:mm:ss")
            let title = isReceived ? "Received message" : "Sent message"
            let color = isReceived ? "blue" : "green"
            title = title + " " + time
            title = "<p style='color:%1'> %2 </p>".arg(color).arg(title)
            textArea.append(title)
            textArea.append(message)
        }

        // 显示提示信息
        function showTip(message) {
            
            textArea.append(message)
        }

        // 根据状态显示提示信息
        function showStatusTip(status) {
            
            switch (status) {
            
            case WebSocket.Connecting:
                showTip("Socket connecting")
                break
            case WebSocket.Open:
                showTip("Socket opened")
                break
            case WebSocket.Closing:
                showTip("Socket closing")
                break
            case WebSocket.Closed:
                showTip("Socket closed")
                break
            case WebSocket.Error:
                showTip("Socket error: " + socket.errorString)
                break
            default:
                break
            }
        }
    }

    ColumnLayout {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 10

        RowLayout {
            
            spacing: 10

            // 服务器 URL 输入框
            TextField {
            
                id: urlField
                text: "wss://ws.postman-echo.com/raw"
                placeholderText: "ws:// or wss://"
            }

            // 连接按钮
            Button {
            
                text: "Connect"
                onClicked: socket.connectToServer(urlField.text)
            }

            // 断开按钮
            Button {
            
                text: "Disconnect"
                onClicked: socket.disconnectFromServer()
            }
        }

        // 消息显示区域
        TextArea {
            
            id: textArea
            Layout.fillWidth: true
            Layout.fillHeight: true
            textFormat: TextEdit.RichText
        }

        RowLayout {
            
            spacing: 10

            // 消息输入框
            TextField {
            
                id: textField
                placeholderText: "message"
            }

            // 点击，发送消息
            Button {
            
                text: "Send"
                onClicked: inner.sendMessage()
            }
        }
    }
}

在界面的最上方，我们创建了一个输入框，用于设置要连接的服务器地址。该地址的默认值是 wss://ws.postman-echo.com/raw，如果想要使用其他地址，可以直接进行更改。在端口设置完成之后，就可以点击右侧的“Connect”按钮连接 WebSocket Server 了。

一旦和服务器连接成功，就可以通过界面最下方的输入框发送消息了。由于我们连接的是一个 Echo Server，所以在消息发送完成之后，服务器也会主动给我们回复相应的信息。

而这个过程中产生的聊天记录，则是由界面中间的 TextArea 实现的。之所以将 textFormat 属性设置为 TextEdit.RichText，是为了实现富文本效果，让发送的消息显示为绿色，接收到的消息显示为蓝色，这样就更容易区分了。

最后，在测试完成之后，可以点击右上方的“Disconnect”按钮来断开连接。还不错吧？有点儿聊天软件的感觉！
