本文介绍了如何在QML中使用WebSocketServer创建WebSocket服务。通过指定端口号、管理客户端连接和处理消息，实现一个简单的Echo Server。文章提供了一个示例，展示了如何控制服务的启动、停止，并演示了消息的回显功能。

在《QML WebSocket 客户端》一节中，我们开发了一个 WebSocket 客户端，但在测试的时候，连接的是 Postman 提供的 Echo Server。现在，是时候编写一个真正属于自己的服务了，而这需要通过 QML 提供的 WebSocketServer 类型来实现。

我们之前说过，QML 中的 WebSocket 类型是基于 C++/Qt 中的 QWebSocket 实现的，而 QWebSocket 和 QWebSocketServer 是成对的，所以正如你想的那样，WebSocketServer 类型是基于 QWebSocketServer 实现的。

也就是说，无论是 WebSocket 客户端编程，还是服务端编程，只要熟悉 C++/Qt 中的相关用法，QML 中的也必然能轻松理解和掌握。

基本步骤

要实现一个 WebSocket Server，需要经过以下步骤。

  1. 导入 WebSocket 模块
import QtWebSockets
  1. 创建 WebSocketServer 对象，并进行配置。

在创建 WebSocketServer 时，需要通过 port 属性来指定要监听的端口号。此外，还应该定义一个 clients 容器（例如：数组）来保存所有的客户端连接，并在 onClientConnected() 事件中将新连接加入到该容器中。

WebSocketServer {
            
    id: server
    port: 8686  // 要监听的端口

    // 客户端连接列表
    property var clients: []

    // 有新客户端连接建立
    onClientConnected: (webSocket)=> {
            
        console.info("Client connected");

        // 将新客户端连接加入列表
        clients.push(webSocket)

        // 收到客户端的消息
        webSocket.textMessageReceived.connect(function(message) {
            
            console.info("Received message: " + message);
            // 进行消息处理，根据需要发送消息给特定客户端（clients）
            // ......
        });

        // 状态发生变化（例如：断开连接）
        webSocket.statusChanged.connect(function(status) {
            
            console.info("Status changed: " + status)
        });
    }

    // 出错时触发（例如：端口被占用）
    onErrorStringChanged: {
            
        console.info("Error: " + errorString)
    }
}

注意： 这里连接了 textMessageReceived() 信号，当收到某个客户端发来的消息时，我们可以遍历 clients，将消息转发给其他客户端，甚至是回显给发送方本身。

  1. 启动服务

为了开始监听连接，我们需要将 listen 属性设置为 true。然后在必要时调用此函数，例如：点击按钮时。

function start() {
            
    server.listen = true
}
  1. 处理消息

由于我们保存了所有的客户端连接 - clients，所以可以在任一时刻将消息推送给任意的客户端。例如，发送文本给所有的客户端：

function sendMessage() {
            
    for (let client of server.clients) {
            
        client.sendTextMessage("Hello, client!")
    }
}
  1. 停止服务

如果想关闭 WebSocket 服务，需要将 listen 属性设置为 false。

function stop() {
            
    // 断开与所有客户端的连接
    for (let client of server.clients) {
            
    	client.active = false
	}
    // 清空客户端连接列表
    server.clients = []

	// 停止监听
    server.listen = false
}

注意： 在关闭服务时，最好断开与客户端的所有连接，这样既能保证资源的正确释放（每个与客户端建立的连接都会分配一定的系统资源，如果不及时断开，可能造成资源泄漏），又能避免出现不可预测的结果（若服务器不主动断开连接，客户端可能会连接异常或者继续发数据，从而导致不可预测的结果）。

使用 WebSocketServer

现在，来编写一个真正属于自己的 Echo Server 吧：

import QtQuick
import QtQuick.Controls
import QtQuick.Layouts
import QtWebSockets

Rectangle {
            
    width: 500; height: 400

    WebSocketServer {
            
        id: server

        // 客户端连接列表
        property var clients: []

        // 有新客户端连接建立
        onClientConnected: (webSocket)=> {
            
            // 将新客户端连接加入列表
            clients.push(webSocket)
            inner.showTip("Client connected")

            // 收到客户端的消息后，进行显示并原封不动的发回去
            webSocket.textMessageReceived.connect(function(message) {
            
                inner.showReceivedMessage(message)
                webSocket.sendTextMessage(message)
            })

            // 状态发生变化时（例如：断开连接），进行提示
            webSocket.statusChanged.connect(function(status) {
            
                if (inner != null)
                    inner.showStatusMessage(webSocket, status)
            })
        }

        // 服务器发生错误时触发（例如：端口被占用）
        onErrorStringChanged: inner.showTip("Error: " + errorString)

        // 开启服务
        function start(port) {
            
            server.port = port
            server.listen = true
        }

        // 停止服务
        function stop() {
            
            // 断开与所有客户端的连接
            for (let client of server.clients) {
            
                client.active = false
            }
            // 清空客户端连接列表
            server.clients = []

            // 停止监听
            server.listen = false
        }
    }

    QtObject {
            
        id: inner

        // 开启服务
        function startService() {
            
            let port = parseInt(portField.text)
            server.start(port)
            showTip("Server listened")
        }

        // 停止服务
        function stopService() {
            
            server.stop()
            showTip("Server closed")
        }

        // 显示客户端发送的消息
        function showReceivedMessage(message) {
            
            let time = Qt.formatDateTime(new Date(), "yyyy-MM-dd hh:mm:ss")
            let title = "Received message " + time
            let color = "blue"
            title = "<p style='color:%1'> %2 </p>".arg(color).arg(title)
            textArea.append(title)
            textArea.append(message)
        }

        // 显示提示信息
        function showTip(message) {
            
            textArea.append(message)
        }

        // 根据和客户端的连接状态显示提示信息
        function showStatusMessage(webSocket, status) {
            
            switch (status) {
            
            case WebSocket.Closing:
                showTip("Socket closing")
                break
            case WebSocket.Closed:
                showTip("Socket closed")
                break
            case WebSocket.Error:
                showTip("Socket error: " + webSocket.errorString)
                break
            default:
                break
            }
        }
    }

    ColumnLayout {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 10

        RowLayout {
            
            spacing: 10

            // 端口输入框
            TextField {
            
                id: portField
                text: "8686"
                placeholderText: "port"
            }

            // 监听按钮
            Button {
            
                text: "Listen"
                onClicked: inner.startService()
            }

            // 关闭按钮
            Button {
            
                text: "Close"
                onClicked: inner.stopService()
            }
        }

        // 消息显示区域
        TextArea {
            
            id: textArea
            Layout.fillWidth: true
            Layout.fillHeight: true
            textFormat: TextEdit.RichText
        }
    }
}

在这个例子中，我们提供了一个界面，用于控制 WebSocket Server 监听的端口号以及服务的启动和停止。端口的默认值是 8686，如果要使用其他值，可以按需更改。在端口设置完成之后，可以点击右侧的“Listen”按钮来启动服务。

一旦服务被启动，就可以连接客户端了。如果部署的是本地服务，那么连接的地址是 ws://localhost:8686 或者 ws://127.0.0.1:8686；如果是远程服务，只需将对应的 IP 改成服务器所在的真实 IP 即可。

至于消息回显，是在 webSocket.textMessageReceived(message) 事件中实现的。当客户端发来消息时，该事件会被立即触发，这时通过调用 webSocket.sendTextMessage(message) 方法，就可以将消息原封不动的发送回至发送者。

为了测试多连接，这里我们同时用了两个 WebSocket 客户端，一个是我们自己开发的，另一个是 Hoppscotch 提供的。可以看出，在服务连接成功之后，它们都可以正常发送消息，并能够自动进行回显。

最后，在测试完成之后，点击右上方的“Close”按钮就可以关闭服务了。
