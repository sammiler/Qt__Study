本文详细介绍了QML中的ListView组件，包括其基本用法、方向设置（垂直和水平布局）、选中和高亮效果、页眉和页脚的添加，以及数据分组和动态操作数据的方法。通过实例展示了如何利用ListView构建可滚动列表，并动态管理数据模型，适用于创建如联系人列表、消息列表等用户界面。

虽然 Repeater 在重复创建多个相似项的时候很方便，但是通常只适用于有限的简单元素，并且它还无法滚动浏览。而基于 Flickable 的视图组件（如 ListView、GridView、TableView、TreeView 等）则弥补了这些缺点，它们不但能够显示大量数据，而且还支持滚动浏览，尤其是针对复杂的布局和交互场景，可以通过分页显示、懒加载、动态更新等手段来提高性能。

就从 ListView 开始吧，它是一个用于显示可滚动列表的组件，允许我们以一种简单而灵活的方式呈现项目，并支持水平和垂直布局。与 Repeater 组件类似，ListView 可以与 model 和 delegate 结合使用，以便动态地显示列表内容并自定义其外观和行为。ListView 在构建基于列表的用户界面时非常有用，如联系人列表、消息列表、菜单等。

下面，我们接着《QML 模型视图》一节中的示例，并以图标视图的形式来显示联系人，看看都涉及 ListView 的哪些用法。

基本用法

先来扩展一下模型中的数据（ContactModel.qml），创建 5 个联系人（包含：姓名、号码、职位、头像等属性）：

import QtQuick

ListModel {
            
    ListElement {
            
        name: "Pony"
        number: "10001"
        position: "CEO"
        portrait: "qrc:/assets/logo.jpg"
    }
    ListElement {
            
        name: "Tony"
        number: "10002"
        position: "CTO"
        portrait: "qrc:/assets/logo2.jpg"
    }
    ListElement {
            
        name: "Kney"
        number: "10003"
        position: "CTO"
        portrait: "qrc:/assets/logo3.jpg"
    }
    ListElement {
            
        name: "Charles"
        number: "10004"
        position: "COO"
        portrait: "qrc:/assets/logo4.jpg"
    }
    ListElement {
            
        name: "Patrick"
        number: "10005"
        position: "COO"
        portrait: "qrc:/assets/logo5.jpg"
    }
}

然后和之前一样，在委托中定义联系人的显示方式（IconDelegate.qml），使得联系人的姓名上方显示一个大头像：

import QtQuick
import QtQuick.Controls
import QtQuick.Layouts

Item {
            
    width: 100; height: 100

    ColumnLayout {
            
        anchors.fill: parent

        Item {
             Layout.fillHeight: true }

        // 头像
        Image {
            
            source: model.portrait
            sourceSize: Qt.size(50, 50)
            Layout.alignment: Qt.AlignCenter
        }

        // 姓名
        Text {
            
            text: model.name
            Layout.preferredHeight: 25
            Layout.preferredWidth: 80
            Layout.alignment: Qt.AlignCenter
            horizontalAlignment: Qt.AlignHCenter
        }

        Item {
             Layout.fillHeight: true }
    }
}

最后，我们用 ListView 将 model 和 delegate 结合起来，并利用 spacing 属性控制委托项之间的间距：

import QtQuick

Rectangle {
            
    width: 120
    height: 400
    color: "lightgray"

    ListView {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 5  // 间隔
        clip: true  // 激活裁剪功能
        model: ContactModel {
            }
        delegate: IconDelegate {
            }
    }
}

由于 ListView 的高度有限，而模型中的联系人较多，因此我们只能看到部分列表内容，而其他更多内容需要滚动浏览。

但是有个问题，ListView 不会限制委托项只在限定的区域内显示。也就是说，委托项可能显示在 ListView 外面。这并不是我们期望的结果，所以为了避免该问题，这里将 clip 属性设置为 true，以激活裁剪功能。

不妨看看对比效果，左侧的 clip 为 false，右侧为 true：

方向设置

默认情况下，ListView 中的列表项是垂直显示的。如果要改变方向，可以通过 orientation 属性来控制，其支持的枚举值如下所示：

枚举 描述
ListView.Horizontal 水平布局列表项
ListView.Vertical （默认值） 垂直布局列表项

注意： 一旦设置为水平布局（ListView.Horizontal），则 ListView 默认会从左到右排列列表项。可以通过 layoutDirection 属性来改变排列方向，Qt.LeftToRight（默认） - 从左到右，Qt.RightToLeft - 从右到左。

现在，我们将联系人在水平方向上排列显示：

import QtQuick

Rectangle {
            
    width: 400
    height: 120
    color: "lightgray"

    ListView {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 5
        clip: true
        orientation: ListView.Horizontal  // 水平布局
        model: ContactModel {
            }
        delegate: IconDelegate {
            }
    }
}

选中和高亮

在上述示例中，当我们点击某个列表项时，不会出现任何选中效果。为了能够标识当前选中项，我们可以为其设置高亮。

先来看看鼠标点击，由于点击动作是委托控制完成的，因此可以在委托中（IconDelegate.qml）添加一个 MouseArea 并响应 clicked() 信号。当鼠标点击时，改变当前的列表项：

import QtQuick
import QtQuick.Controls
import QtQuick.Layouts

Item {
            
    id: wrapper
    width: 100; height: 100

    ColumnLayout {
            
        anchors.fill: parent

        Item {
             Layout.fillHeight: true }

        // 头像
        Image {
            
            source: model.portrait
            sourceSize: Qt.size(50, 50)
            Layout.alignment: Qt.AlignCenter
        }

        // 姓名
        Text {
            
            text: model.name
            Layout.preferredHeight: 25
            Layout.preferredWidth: 80
            Layout.alignment: Qt.AlignCenter
            horizontalAlignment: Qt.AlignHCenter
        }

        Item {
             Layout.fillHeight: true }
    }

    // 鼠标事件
    MouseArea {
            
        anchors.fill: parent
        // 单击时，改变当前选项（可通过 index 属性访问 model 中元素索引）
        onClicked: {
            
            wrapper.ListView.view.currentIndex = model.index
        }
    }
}

而高亮效果需要用到两个属性，首先要将 focus 属性为 true，让 ListView 能够获得焦点；然后是设置 highlight 属性，以指定高亮的委托元素。

下面这个高亮效果，是一个淡蓝色的圆角矩形：

import QtQuick

Rectangle {
            
    width: 120
    height: 650
    color: "lightgray"

    ListView {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 5
        clip: true
        model: ContactModel {
            }
        delegate: IconDelegate {
            }

        // 能够获取焦点
        focus: true
        // 指定高亮委托元素
        highlight: Rectangle {
            
            color: "lightsteelblue"
            radius: 5
        }
    }
}

此外，还有一些属性可以控制高亮的行为，例如：highlightMoveDuration（高亮移动持续时间） - 定义了列表项获取焦点时从当前位置移到期望位置所需的时间，通过设置这个属性，可以控制高亮移动的速度；highlightMoveVelocity（高亮移动速度） - 影响了在列表项获取焦点时的移动速度，它可以通过设定来改变高亮移动的快慢程度。

页眉和页脚

为了突出开始和结尾，ListView 还支持插入页眉和页脚，这主要由 header 和 footer 属性控制的。它们可以包含任何类型的 QML 内容，因此可以自由地定制样式和布局。较常见的用法是，用它们显示一些额外的信息或者操作，例如：标题、摘要、导航按钮等。

在下面的例子中，我们设置了一个文本为“Contacts”的页眉，并在页脚处显示了联系人的数量：

import QtQuick

// 4. 页眉和页脚
Rectangle {
            
    width: 120
    height: 650
    color: "lightgray"

    ListView {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 5
        clip: true
        model: ContactModel {
            }
        delegate: IconDelegate {
            }

        // 页眉和页脚
        header: headerComponent
        footer: footerComponent
    }

    // 页眉组件
    Component {
            
        id: headerComponent
        Text {
            
            width: ListView.view.width
            height: 40
            horizontalAlignment: Text.AlignHCenter
            verticalAlignment: Text.AlignVCenter
            text: "Contacts"
            font.bold: true
            font.pixelSize: 18
        }
    }

    // 页脚组件
    Component {
            
        id: footerComponent
        Text {
            
            width: ListView.view.width
            height: 30
            color: "gray"
            horizontalAlignment: Text.AlignHCenter
            verticalAlignment: Text.AlignVCenter
            text: "Count: " + ListView.view.model.count
        }
    }
}

数据分组

ListView 非常灵活，它不但能够进行平面显示，而且还可以使用 section 属性进行分组显示。分组的用途比较多，比如按照物品的类别分组（水果、蔬菜）、按照时间属性分组（年份、季节）、按照地理位置分组（国家、城市）等。

回到前面的数据模型（ContactModel.qml），可以看到每个联系人都有 position 属性，我们按照职位将他们分为了 CEO、CTO、COO，来看看分组后的效果：

import QtQuick

Rectangle {
            
    width: 120
    height: 650
    color: "lightgray"

    ListView {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 5
        clip: true
        model: ContactModel {
            }
        delegate: IconDelegate {
            }

        // 将相关项组合成一个组
        section {
            
            property: "position"  // 该属性决定使用哪个属性
            criteria: ViewSection.FullString
            delegate: sectionComponent
        }
    }

    // 分组委托组件
    Component {
            
        id: sectionComponent
        Rectangle {
            
            color: "lightsteelblue"
            width: ListView.view.width
            height: 30
            Text {
            
                anchors.fill: parent
                anchors.margins: 5
                font.bold: true
                text: section  // 访问当前分组名称
                horizontalAlignment: Text.AlignLeft
                verticalAlignment: Text.AlignVCenter
            }
        }
    }
}

其中，section.property 决定了使用哪个属性进行分组，这里就是上面所说的联系人职位 - position。而 section.criteria 是基于 section.property 形成的分组标准，枚举取值如下：

枚举 描述
ViewSection.FullString（默认值） 分组是基于 section.property 创建的值
ViewSection.FirstCharacter 分组是基于 section.property 的首字母创建的值（例如，联系人按照姓氏 A、B、C 分组）

至于分组的委托组件，则是由 section.delegate 指定的。它的作用是创建分组标题，并在列表项之前插入分组。上面，我们用 sectionComponent 作为分组委托，这是一个淡蓝色矩形，在矩形内部，我们定义了一个 Text 文本元素，并通过 section 绑定属性显示了当前分组的名称。

动态操作

目前为止，我们的界面数据都是静态的，然而在实际应用过程中，通常会涉及到对界面数据的动态操作，包括增加新数据、删除现有数据、修改数据内容以及查询特定数据等。

由于我们使用了 ListModel 来管理 ListView 中的数据源，因此可以通过添加、移除、修改和查询 ListModel 中的数据项来实现这些操作。为此，ListModel 提供了相应的便捷方法：

  • 增加：使用 append() 或者 insert() 方法向数据模型中添加新数据项
  • 删除：使用 remove() 方法从数据模型中删除特定位置的数据项
  • 修改：使用 set() 或者 setProperty() 方法来修改特定位置的数据项
  • 查询：通过 get() 方法获取特定位置的数据项

下面是一个包含了增删改查的完整示例，在这个例子中，我们通过单击列表项来查询联系人的信息，通过双击列表项来更新联系人的姓名，通过点击添加按钮来新增一个联系人，通过点击删除按钮移来除当前选中的联系人。

由于涉及到单击和双击操作，因此需要在委托中响应 MouseArea 的 clicked() 和 doubleClicked() 信号。为了和前面的委托区分开，这里我们新创建一个名为 EditIIconDelegate.qml 的委托：

import QtQuick
import QtQuick.Controls
import QtQuick.Layouts

Item {
            
    id: wrapper
    width: 100; height: 100

    ColumnLayout {
            
        anchors.fill: parent

        Item {
             Layout.fillHeight: true }

        // 头像
        Image {
            
            source: model.portrait
            sourceSize: Qt.size(50, 50)
            Layout.alignment: Qt.AlignCenter
        }

        // 姓名
        Text {
            
            id: displayText
            text: model.name
            Layout.preferredHeight: 25
            Layout.preferredWidth: 80
            Layout.alignment: Qt.AlignCenter
            horizontalAlignment: Qt.AlignHCenter
        }

        // 姓名编辑框
        TextField {
            
            id: textField
            text: model.name
            Layout.preferredHeight: 25
            Layout.preferredWidth: 80
            Layout.alignment: Qt.AlignCenter
            horizontalAlignment: Qt.AlignHCenter
            // 编辑框的显示状态取决于姓名显示框（状态相反）
            visible: !displayText.visible
            // 3. 更新联系人（编辑完成更新）
            onEditingFinished: {
            
                model.name = textField.text
                textField.focus = false
                displayText.visible = true
            }
        }

        Item {
             Layout.fillHeight: true}
    }

    // 鼠标事件
    MouseArea {
            
        anchors.fill: parent
        // 单击时，改变当前选项
        onClicked: wrapper.ListView.view.currentIndex = model.index
        // 双击时，进入编辑模式
        onDoubleClicked: {
            
            displayText.visible = false
            textField.focus = true
            textField.forceActiveFocus()
        }
    }
}

为了实现双击编辑联系人，我们添加了一个 TextField 编辑框，其默认状态是隐藏的。当用户双击列表项时，它会立即显示出来并进入编辑模式。一旦编辑完成，就会自动触发 editingFinished() 信号，这时会隐藏编辑框，并将联系人的姓名更新为输入框中的内容。

而对于联系人的单击查询，委托的任务只负责单击时更改当前选项 currentIndex。至于选项变化后的事情，则可以通过响应 ListView 的 onCurrentIndexChanged() 事件来实现。在该事件中，我们通过 model.get(currentIndex) 获取了当前选中的联系人对象，并将该联系人的姓名和号码显示在了页脚上：

import QtQuick
import QtQuick.Controls
import QtQuick.Layouts

Rectangle {
            
    width: 120
    height: 700

    ColumnLayout {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 10

        Item {
             Layout.fillHeight: true }

        Rectangle {
            
            Layout.preferredWidth: 120
            Layout.preferredHeight: 650
            color: "lightgray"

            ListView {
            
                id: listView
                anchors.fill: parent
                anchors.margins: 10
                spacing: 5
                clip: true
                model: ContactModel {
             id: contactModel }
                delegate: EditIIconDelegate {
            }

                focus: true
                highlight: Rectangle {
             color: "lightsteelblue"; radius: 5 }

                // 页脚
                footer: footerComponent

                // 4. 查询联系人（当前项改变时，刷新页脚显示）
                onCurrentIndexChanged: {
            
                    var item = model.get(currentIndex)
                    footerItem.text = item.name + ":" + item.number
                }
            }

            // 页脚
            Component {
            
                id: footerComponent
                Text {
            
                    width: ListView.view.width
                    height: 30
                    color: "gray"
                    horizontalAlignment: Text.AlignHCenter
                    verticalAlignment: Text.AlignVCenter
                }
            }
        }

        RowLayout {
            
            id: btnLayout
            spacing: 5

            property int index: 1

            Item {
             Layout.fillWidth: true }

            // 1. 新增联系人
            RoundButton {
            
                text: "\u002B"
                onClicked: {
            
                    var contact = {
            
                        name: "Waleon" + btnLayout.index,
                        number: "10000",
                        position: "COO",
                        portrait: "qrc:/assets/logo.jpg"
                    }
                    contactModel.append(contact)
                    btnLayout.index++
                }
            }

            // 2. 删除联系人
            RoundButton {
            
                text: "\u002D"
                onClicked: {
            
                    if (contactModel.count > 0)
                        contactModel.remove(listView.currentIndex)
                }
            }

            Item {
             Layout.fillWidth: true }
        }

        Item {
             Layout.fillHeight: true }
    }
}

最后，我们还定义了两个按钮。当点击按钮“+”时，会创建一个名称自增的联系人，并通过 ListModel 的 append() 方法将其添加至联系人列表中。相反地，当点击按钮“-”时，会通过 ListModel 的 remove() 方法删除当前选中的联系人。
