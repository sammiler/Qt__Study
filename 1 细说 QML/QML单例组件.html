本文介绍了如何在QML中创建和使用单例组件，以方便全局数据共享和状态管理。通过定义单例组件、调用qmlRegisterSingletonType()方法或使用qmldir文件注册单例类型，以及导入并使用单例组件的步骤，详细阐述了QML单例组件的工作原理和实践方法。

在使用 QML 开发时，我们经常需要自定义组件，来实现一些通用的 UI 元素（如导航栏、圆形进度条等）。这样做的好处是，可以将复杂的界面逻辑封装到一个独立的组件中，从而提高代码的可维护性和可复用性。

虽然这种普通的自定义组件很好用，但是并不利于数据共享以及全局状态的管理（如全局主题样式、全局配置、全局数据缓存等）。这时，单例组件了就派上用场了，通过它我们可以很方便地共享和管理应用级别的信息。

要使用单例组件，需经过以下几个步骤：

  1. 在 QML 脚本中使用 pragma Singleton 关键字，定义单例组件；
  2. 调用 qmlRegisterSingletonType() 方法或者使用 qmldir 文件，将组件注册为单例类型；
  3. 通过 import 导入所定义的模块或者目录，然后使用单例。

定义单例组件

假设，Constants.qml 是我们自定义的组件，里面包含了一些全局常量。要将其定义为单例，需要在文件的顶部添加关键字 pragma Singleton：

pragma Singleton

import QtQuick

QtObject {
            
    readonly property string appName: "xxx App"
    readonly property color backgroundColor: "green"
    readonly property string logo: "qrc:/images/logo.jpg"
}

注册单例类型

组件有了之后，现在需要做的是将其注册为单例类型，为了实现这一点，有以下两种可选方式。

调用 qmlRegisterSingletonType() 方法

如果想在 C++ 中注册单例，可以调用下述方法：

int qmlRegisterSingletonType(const QUrl &url, const char *uri, int versionMajor, int versionMinor, const char *qmlName)

  • url：要注册的组件，也就是上面定义的 qml 文件
  • uri：标识符，在被导入时会被用到
  • versionMajor：主版本号
  • versionMinor：次版本号
  • qmlName：注册的单例类型的名字
#include <QGuiApplication>
#include <QQmlApplicationEngine>

int main(int argc, char *argv[])
{
            
    QGuiApplication app(argc, argv);

    // 注册单例类型
    qmlRegisterSingletonType(QUrl("qrc:/qml/global/Constants.qml"), "qml.tutorial.singleton", 1, 0, "Constants");

    QQmlApplicationEngine engine;
    engine.load(QUrl("qrc:/qml/main.qml"));
    if (engine.rootObjects().isEmpty())
        return -1;

    return app.exec();
}

倘若创建的是 Qt Quick UI 项目（即：没有 C++），那么这种方法就行不通了，这时我们可以采用下面的方式。

使用 qmldir 文件

qmldir 文件是一个纯文本文件，它的主要作用之一是“对象类型声明”，语法格式如下：

[singleton] <TypeName> <InitialVersion> <File>

我们要做的是，在 Constants.qml 所在目录下（例如：global）创建一个 qmldir 文件，并通过上述语法将组件注册为单例类型：

singleton Constants 1.0 Constants.qml  
CircularBar 1.0 CircularProgressBar.qml

最终的目录结构，应该是这样的：

|  + global
|  |  + qmldir
|  |  + Constants.qml
|  |  + CircularProgressBar.qml
|  |  + NavigationBar.qml
|  + main.qml

当导入 global 目录时，QML 引擎会查找其中的 qmldir 文件，并通过它来导入该目录中的 QML 脚本。即使某个脚本没在 qmldir 中声明，仍然也会被导入，但使用的是默认行为（即：非单例组件，并将文件名作为组件名），例如：NavigationBar.qml 会被导入并注册，并使用 NavigationBar 作为组件名。

另外，也可以使用 qmldir 为组件定义一个别名，例如：CircularProgressBar.qml 会被注册为 CircularBar。这时在创建元素时，既可以使用文件名 CircularProgressBar，也可以使用别名 CircularBar。

导入并使用单例

如果选择了 qmlRegisterSingletonType() 方式，则必须导入所定义的模块（该函数的第二个参数 uri），然后才能使用注册的单例类型（第五个参数 qmlName）：

import QtQuick
import qml.tutorial.singleton 1.0

Rectangle {
            
    color: Constants.backgroundColor
}

如果使用的是 qmldir，那么只需要导入目录即可，这时将注册该目录中的所有脚本：

import QtQuick
import "qrc:/qml/global/"

Rectangle {
            
    color: Constants.backgroundColor
}


可以看出，矩形的背景变为了绿色，这说明单例导入成功并能正常使用了。
