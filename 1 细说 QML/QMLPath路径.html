在 QML 中，Path 是一种用于定义和描述路径的元素，通常需要和 Canvas、PathView、Shape 等元素结合一起使用，用于生成复杂的图形或动画效果。

所谓路径，是由一系列路径元素组成的集合，从起点开始，各段首尾相连，最终到达终点结束。其中，每个路径元素代表着整个路径中的一段，可以是直线、曲线或者其他形状。

以下是 Path 元素提供的一些属性：

属性 描述
startX 路径起点的 x 坐标
startY 路径起点的 y 坐标
pathElements 路径元素列表，保存了组成路径的多个路径元素，常见的路径元素有：PathLine、PathQuad、PathCubic、PathArc、PathAngleArc、PathCurve、PathSvg 等。
closed 路径的终点是否与起点重合，若重合，则为 true；否则为 false。

值得注意的是，Path 并没有提供类似 endX / endY 这样的属性来指定路径的终点，路径的终点是由路径上最后一个元素定义的。而中间的路径元素则依次连接起点和终点，从而形成一条完整的路径。

尽管 Path 表示一条路径，但其本身是不可见的。为了直观地了解和学习它，接下来，我们将通过 Canvas 绘制各种基本路径以及一些复杂的路径。

基本路径

PathLine（直线）

PathLine 定义了路径中的一段直线，连接起点和终点生成直线段。

PathLine 的起点是上一个路径元素的终点（或 startX / startY），而终点由 x / y（或者 relativeX / relativeY）属性定义。通过调整这些属性，可以控制直线的长度和方向，从而实现不同的直线段效果。

顺便解释一下：

  • x / y：绝对坐标，表示路径的终点相对于父元素（例如：PathView）的坐标位置。
  • relativeX / relativeY：相对坐标，表示路径的终点相对于路径的起点（例如：startX / startY）的偏移量。

一个简单的直线如下所示：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        startX: 0; startY: root.height/2

        // 绝对坐标
        PathLine {
             x: root.width; y: root.height/2 }

        // 相对坐标
        // PathLine { relativeX: root.width; relativeY: 0 }

        // 一个绝对坐标，一个相对坐标
        // PathLine { x: root.width; relativeY: 0 }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

该路径是一条位于界面中间的水平直线，对于这条直线的终点定义，我们可以像例子中那样使用绝对坐标，也可以使用相对坐标，甚至还可以将它们混合在一起使用（即：一个绝对坐标，一个相对坐标）。

PathQuad（二次贝塞尔曲线）

PathQuad 定义了路径中的一段二次贝塞尔曲线，由起点、控制点和终点三个点组成，其中控制点（即：形状调节器）用于控制曲线的形状。

与 PathLine 类似，PathQuad 的起点也是上一个路径元素的终点（或 startX / startY），而终点由 x / y（或者 relativeX / relativeY）属性定义，控制点由 controlX / controlY（或者 relativeControlX / relativeControlY）属性定义。通过调整这些属性，可以改变曲线的形状，从而创建各种曲线效果。

原理图如下（来自 Wiki）：

一个简单的二次贝塞尔曲线如下所示：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        startX: 0; startY: root.height / 2
        PathQuad {
            
            x: root.width; y: root.height / 2
            controlX: root.width / 2; controlY: 0
        }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

这个样式正是照着原理图的效果实现的，该二次贝塞尔曲线的起点为 (0, root.height / 2)，终点为 (root.width, root.height / 2)，控制点为 (root.width / 2, 0)。

PathCubic（三次贝塞尔曲线）

PathCubic 定义了路径中的一段三次贝塞尔曲线，由起点、两个控制点和终点四个点组成。由于比 PathQuad 多了一个控制点，因此 PathCubic 提供了更高的灵活性和控制性。

同样地，PathCubic 的起点也是上一个路径元素的终点（或 startX / startY），而终点由 x / y（或者 relativeX / relativeY）属性定义，两个控制点分别由 control1X / control1Y（或者 relativeControl1X / relativeControl1Y）和 control2X / control2Y（或者 relativeControl2X / relativeControl2Y）属性定义。

原理图如下（来自 Wiki）：

一个简单的三次贝塞尔曲线如下所示：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        startX: 0; startY: 100
        PathCubic {
            
            x: 200; y: 100
            control1X: 0; control1Y: 0
            control2X: 150; control2Y: 0
        }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

该三次贝塞尔曲线的起点为 (0, 100)，终点为 (200, 100)，控制点一为 (0, 0)，控制点二为 (150, 0)。如果有什么不明白，记得多看看原理图。

PathArc（圆弧）

PathArc 定义了路径中的一段圆弧，由起点、圆弧半径、终点以及圆弧方向等参数定义。

PathArc 的起点也是上一个路径元素的终点（或 startX / startY），终点由 x / y（或者 relativeX / relativeY）属性定义，弧线的半径由 radiusX / radiusY 属性定义，direction 属性则定义了圆弧的方向，可取值有 PathArc.Clockwise（默认值）和 PathArc.Counterclockwise，分别表示顺时针和逆时针方向。

当指定了起点、终点、半径和方向之后，有可能存在两条弧线满足条件，这时可以使用 useLargeArc 属性来进行区分，若使用默认值 false ，则取较小的弧线；若将值设置为 true，则取较大的弧线。

原理图如下：

为了演示这一点，我们分别绘制两个不同大小的圆弧：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        startX: 0; startY: 100
        PathArc {
            
            id: arc
            x: 100; y: 200
            radiusX: 100; radiusY: 100
            direction: PathArc.Clockwise
        }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"

        // 较小的圆弧
        context.path = myPath
        context.stroke()

        // 较大的圆弧
        arc.useLargeArc = true
        context.path = myPath
        context.stroke()
    }
}

可以看到，这两个圆弧是由同一个 Path 绘制出来的，也就是说，它们的起点、终点、半径和方向完全一样。通过为 useLargeArc 分配不同的值，我们先后绘制出了较小的圆弧和较大的圆弧。

PathAngleArc（圆弧）

PathAngleArc 类似于 PathArc，也是用于定义圆弧路径的。不同的是，PathAngleArc 是基于角度来定义圆弧路径，而 PathArc 则是基于起点和终点坐标定义的。

圆弧的中心点是由 centerX / centerY 属性定义的，X / Y 轴半径由 radiusX / radiusY 属性定义，startAngle 属性则指定了圆弧的起始角度（3 点钟位置为零度，顺时针为正），sweepAngle 属性是圆弧从 startAngle 开始扫过的角度（顺时针为正，360 度是一个完整的圆）。

moveToStart 属性指定了该元素是否应该与前一个路径元素（或 startX / startY）断开连接，默认值为 true，如果设置为 false，则前一个路径元素的终点（或 startX / startY）将用一条直线连接到弧的起点。

原理图如下：

一个简单的圆弧如下所示：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        PathAngleArc {
            
            centerX: 0; centerY: 0
            radiusX: 150; radiusY: 150
            startAngle: 20
            sweepAngle: 50
        }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

这个圆弧路径很简单，中心点为 (0, 0)，X 轴和 Y 轴半径均为 150，起始角度为 20 度，扫过的角度为 50 度。

PathCurve（曲线）

PathCurve 定义了路径中的一段曲线，通常使用多个点来生成。相比 PathQuad，这种方式更加简单，而且曲线也更加平滑。

PathCurve 的起点也是上一个路径元素的终点（或 startX / startY），终点由 x / y（或者 relativeX / relativeY）属性定义。

一个简单的曲线如下所示：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        startX: 0; startY: 100

        PathCurve {
             x: 50; y: 50 }
        PathCurve {
             x: 100; y: 150 }
        PathCurve {
             x: 150; y: 50 }
        PathCurve {
             x: 200; y: 100 }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

这条曲线看起来像山脉一样，共由五个点组成。除了起点由 startX 和 startY 表示外，还有四个点由 PathCurve 创建。通过连接这些点，形成了一条平滑的曲线。

PathSvg（SVG）

PathSvg 允许使用 SVG 格式的路径数据来定义路径

通过指定 path 属性，可以将 SVG 格式的路径数据直接导入到 QML 中，实现复杂的路径形状、曲线和闭合效果。

一个简单的 SVG 路径如下：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        startX: 50; startY: 50
        PathSvg {
             path: "L 150 50 L 100 150 z" }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

这是一个三角形路径，startX 和 startY 指定了路径的起点坐标为 (50, 50)，PathSvg 中的 path 属性使用 SVG 格式描述了路径数据，各部分含义如下：

  • “L 150 50”：是一个路径命令，表示从当前点画一条直线到指定的坐标点 (150, 50)，“L” 代表直线命令，后面跟着目标点的坐标。
  • “L 100 150”：同样是一个直线命令，表示从上一个点 (150, 50) 画一条直线到指定的坐标点 (100, 150) 。
  • “z”：闭合路径命令，表示将当前点与起始点连接起来，形成一个封闭的路径。

因此，整体含义是定义了一个起点为 (50, 50)，包含一条从 (50, 50) 到 (150, 50) 的直线段，再从 (150, 50) 到 (100, 150) 的直线段，最后闭合路径与起点 (50, 50) 相连，形成一个封闭的三角形。

复杂路径

复杂路径是将多个基本路径组合在一起，从而创建出更加复杂和多样化的路径形状。

比如，可以通过连接多条 PathLine 创建复杂的折线，或者连接多个 PathArc / PathAngleArc 创建复杂的弧线，甚至还可以将不同的路径元素连接起来组成更复杂的路径。

例如，下面的这个拱桥形状就是一条较为复杂的路径：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath
        startX: 0
        startY: 100

        PathLine {
             x: 25; y: 100 }
        PathArc {
            
            x: 75; y: 100
            radiusX: 25; radiusY: 25
        }
        PathLine {
             x: 125; y: 100 }
        PathArc {
            
            x: 175; y: 100
            radiusX: 25; radiusY: 25
        }
        PathLine {
             x: 200; y: 100 }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

这个路径不难理解，它是由三条直线和两个圆弧组成的。虽然稍显复杂，但一旦掌握了基本路径的概念，即使是更复杂的路径，也可以迅速组合完成。

移动路径位置

默认情况下，当前路径元素的起点是上一个路径元素的终点（或 startX / startY），所以我们看到的效果是路径元素之间首尾相连。倘若两个路径元素需要分隔开来，那么该怎么办呢？

这时就需要用到 PathMove 了，它可以快速改变路径的位置，通过指定 x / y（或者 relativeX / relativeY）属性即可。

我们以面部表情为例，来实现一个简单的笑脸：

import QtQuick

Canvas {
            
    id: root
    width: 200; height: 200
    contextType: "2d"

    Path {
            
        id: myPath

        // 左眼
        startX: 25; startY: 100
        PathQuad {
            
            x: 75; y: 100
            controlX: 50; controlY: 50
        }

        // 右眼
        PathMove {
             x: 125; y: 100 }
        PathQuad {
            
            x: 175; y: 100
            controlX: 150; controlY: 50
        }

        // 嘴巴
        PathMove {
             x: 90; y: 150 }
        PathQuad {
            
            x: 110; y: 150
            controlX: 100; controlY: 175
        }
    }

    onPaint: {
            
        context.lineWidth = 1
        context.strokeStyle = "black"
        context.path = myPath
        context.stroke()
    }
}

这里，我们首先使用 startX 和 startY 定义了路径的起始点，然后通过 PathQuad 创建了一个二次贝塞尔曲线来绘制左眼；接着利用 PathMove 将路径的当前点移动到右眼的起始点，并再次使用 PathQuad 创建了一个二次贝塞尔曲线来绘制右眼；最后，以相似的方式将路径的当前点移动到嘴巴的起始点，再次创建了一个二次贝塞尔曲线来绘制嘴巴。这样，就完成了整个笑脸的绘制过程。

是不是蛮有意思的，如果对绘画感兴趣，建议大家研究一下简笔画哦！
