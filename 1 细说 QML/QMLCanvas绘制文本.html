关于 QML Canvas，我们已经学习了如何绘制基本图形、应用样式和颜色、绘制图像等，现在来看一下如何在 Canvas 中绘制文本。

Canvas 提供了两种方式来渲染文本：

方法 描述
object fillText(text, x, y) 在指定的 (x, y) 位置处填充文本
object strokeText(text, x, y) 在指定的 (x, y) 位置处绘制文本的边框

当然了，也可以利用 fill() 和 stroke() 将两者组合起来，这样既可以给文本填充颜色，又可以给文本绘制边框。

完整的思维导图如下：

绘制文本

我们先来看最简单的用法，在画布的 (10, 100) 位置处绘制文本：

import QtQuick

Canvas {
            
    width: 200; height: 200

    onPaint: {
            
        var ctx = getContext("2d")

        // 填充文本
        ctx.fillText("Hello, Canvas!", 10, 100)
    }
}

你会发现，文本比较小，不是很清晰，这是因为默认的字体是“10px sans-serif”，所以看起来有点模糊。

设置字体

为了让文字清晰一些，我们使用 font 属性设置一下字体，顺便对比一下各字体的大小及清晰度：

Canvas {
            
    width: 300; height: 500

    onPaint: {
            
        var ctx = getContext("2d")

        // 字体
        var fontFamily = "px sans-serif"

        for (var i = 10; i <= 24; i++) {
            
            ctx.font = i + fontFamily;
            ctx.fillText(i + "px: " + "Hello, Canvas!", 10, 30*(i-9));
        }
    }
}

这里，我们利用 for 循环绘制一组 10px ~ 24px 的文本，它们绘制点的 x 坐标都是 10，y 坐标是根据 30*(i-9) 动态计算出来的，这样所有文字就左侧对齐了，且文字间能保持一定的间距 30px，看起来会美观一些。

怎么样，是不是字体越大越清晰。所以呢，选择合适的字体很重要，这也是为什么网页中普遍使用 14px+ 的原因了。

设置样式

绝大多数情况下，我们见到的文本和上面的一样，都是实心黑色的，如果想更改文本色或者给文本描边（即：边框），该怎么办呢？

下面的例子，我们分别绘制 3 个不同样式的文本 - 绿色实心文本，描边为红色的文本，被绿色填充、并带有红色描边的文本：

Canvas {
            
    width: 250; height: 200

    onPaint: {
            
        var ctx = getContext("2d")

        ctx.font = "36px sans-serif"

        // 填充文本
        ctx.fillStyle = "green"
        ctx.fillText("Fill", 10, 50)

        // 文本描边（无填充）
        ctx.lineWidth = 1
        ctx.strokeStyle = "red"
        ctx.strokeText("Stroke", 10, 100)

        // 填充文本并描边
        ctx.text("Fill & Stroke", 10, 150)
        ctx.fill()
        ctx.stroke()
    }
}

是不是很简单？和矩形的绘制原理是一模一样的。

  • 仅填充： fillStyle 设置填充色，并使用 fillXXX() 填充；
  • 仅描边： lineWidth 和 strokeStyle 设置画笔宽度和颜色，并使用 strokeXXX() 绘制边框；
  • 填充并描边： fillStyle 设置填充色，lineWidth 和 strokeStyle 设置画笔宽度和颜色，并同时使用 fill() 和 stroke()。

计算文本宽度

有时候，在绘制文本之前，我们可能想事先知道它的宽度，比如：给文本绘制背景。这时可以使用 measureText() 方法，它返回了一个具有 width 属性的对象，相当于调用了 QFontMetrics::horizontalAdvance()，能快速地预测出当前字体下的文本宽度。

我们来计算一下，当字体为“16px sans-serif”时，文本“Hello, Canvas!”的宽度：

Canvas {
            
    width: 200; height: 200

    onPaint: {
            
        var ctx = getContext("2d")

        ctx.font = "16px sans-serif"

        // 预测文本宽度
        var measure = ctx.measureText("Hello, Canvas!");
        ctx.fillText("'Hello, Canvas!' width:" + measure.width, 10, 100);
    }
}

可以看出，文本的宽度是 101px，使用起来非常方便。

文本基线

不知道你注意过没有，上面我们在绘制文本时，都不是从原点 (0, 0) 开始的，这是为什么呢？因为不这么做，就会出问题！

不信来看看下面这个例子：

Canvas {
            
    width: 200; height: 100

    onPaint: {
            
        var ctx = getContext("2d")

        ctx.font = "16px sans-serif"
        ctx.fillText("Hello, Canvas!", 0, 0)
    }
}

文字只能看见一点点，好像消失了一样。如果想知道具体原因，就需要了解一个概念 - textBaseline（文本基线，可以理解为参照物）。

回想一下，在学习 26 个英文字母时，是不是有“四线三格”来标准它们的写法。类似的，在 Canvas 绘制文本时，也有这么一张图，只不过是“六线五格”，是用来指定文本和绘制点之间的关系的。

值 描述
top 文本基线在文本的顶部
hanging 文本基线是悬挂基线
middle 文本基线在文本的中间
alphabetic 文本基线是标准的字母基线
ideographic 文本基线是表意字基线。如果字符本身超出了 alphabetic 基线，那么 ideograhpic 基线位置在字符本身的底部。
bottom 文本基线在文本的底部。与 ideographic 基线的区别在于，ideographic 基线不需要考虑下行字母

有了这张图，理解起来就简单多了。由于绘制点指定了基线的起始点位置，因此在上面的示例中，当基线为 alphabetic 时（默认值），意味着调用 fillText() 会使得 alphabetic 的起始位置是 (0, 0)，所以就导致文本的顶部看不见了。

现在，我们将基线设置为 top，再来看看绘制效果：

Canvas {
            
    width: 200; height: 100

    onPaint: {
            
        var ctx = getContext("2d")

        ctx.font = "16px sans-serif"
        // 设置基线在文本的顶部
        ctx.textBaseline = "top"
        ctx.fillText("Hello, Canvas!", 0, 0)
    }
}

不出意外，文本完全地显示出来了。

为了更好地理解基线对绘制文本的影响，我们再举一个例子。画一条蓝线，看绘制点与基线的位置究竟是如何的：

Canvas {
            
    width: 450; height: 200

    onPaint: {
            
        var ctx = getContext("2d")

        ctx.font = "16px sans-serif"

        // 基线（默认值：alphabetic）
        var textBaseline = ['top', 'hanging', 'middle', 'alphabetic', 'ideographic', 'bottom']

        // 画一条蓝线（以确定绘制点与基线的位置）
        ctx.strokeStyle = "blue"
        ctx.moveTo(0, 100)
        ctx.lineTo(450, 100)
        ctx.stroke()

        var x = 10

        for (var i = 0; i < textBaseline.length; i++) {
            
            ctx.textBaseline = textBaseline[i]
            ctx.fillText(textBaseline[i], x, 100)

            // 根据当前文本的宽度，计算下一个文本的 x 坐标
            var measure = ctx.measureText(textBaseline[i])
            x = x + measure.width + 10
        }
    }
}

这下是不是清晰多了，同样的 y 坐标，当 textBaseline 取值不同时，文本的位置也截然不同。

水平对齐

如果将 textBaseline 看做是文本在垂直方向上的对齐，那么水平方向上也同样可以设置对齐，这时就需要介绍另一个属性了 - textAlign，其取值包含：

值 描述
start 文本在指定的位置开始（默认值）
end 文本在指定的位置结束
center 文本的中心被放置在指定的位置
left 文本左对齐
right 文本右对齐

这里就不再详细举例子了，和上面一样，我们直接绘制一条直线，看看每种 textAlign 属性值的效果：

Canvas {
            
    width: 200; height: 200

    onPaint: {
            
        var ctx = getContext("2d")

        // 水平对齐方式（默认值：start）
        var textAlign = ['left', 'right', 'center', 'start', 'end']

        // 绘制一条“基准线”
        ctx.strokeStyle = "blue"
        ctx.moveTo(100, 0)
        ctx.lineTo(100, 200)
        ctx.stroke()

        // 绘制文本
        ctx.font = "16px sans-serif"

        for (var i = 0; i < textAlign.length; i++) {
            
            ctx.textAlign = textAlign[i]
            ctx.fillText(textAlign[i], 100, 40*(i+1))
        }
    }
}

需要注意的是，在大多数情况下，start/end 和 left/right 的效果相同。但在一些特殊的场景中（如中英文混排），使用 start 会根据文本的实际方向来判断水平对齐的位置，而 left 则表示文本水平方向左对齐，与实际文本方向可能有偏差。因此，为了更加符合文本实际的水平排列效果，建议使用 start/end，避免出现与文本混排情况不匹配的情况。
