本文介绍了QML中状态的概念，通过交通信号灯的例子展示了如何创建、切换和扩展状态，以及利用状态改变来更改组件的属性、父对象、锚点和执行脚本，帮助读者深入理解QML状态驱动的UI设计。

在设计用户界面时，经常会用到状态驱动的方式，使界面根据当前的状态的不同而呈现不同的内容。以交通信号灯为例，它有行驶、警示和停止三种状态。在行驶状态下，绿灯亮起，指示所有车辆可以通行；在警示状态下，黄灯亮起，提示司机注意并准备停车；而在停止状态下，红灯亮起，提示所有车辆停止前行。这种状态驱动的设计使用户能够清晰地了解当前情况，从而更好地作出相应的行动。

在 QML 中，状态是由 State 元素定义的，每个状态都有一个唯一的名称（name），以及对应的属性或样式的变化（changes）。通过定义不同的状态，我们可以很方便地控制 UI 组件的显示和隐藏，展示不同的可用操作，启动或停止动画，以及执行一些脚本等。

所有基于 Item 的对象都有 states 和 state 属性，其中 states 属性保存了该 Item 所有可能的状态（即：多个 State 对象），而 state 属性指定了 Item 的当前状态。若要更改 Item 的状态，只需将 state 属性设置为对应的状态名称即可。

完整的思维导图如下：

创建状态

在创建 Item 的状态时，我们需要为每一种状态定义一个 State 对象，并将它们添加到 states 属性的状态列表中。然后，可以从中选择感兴趣的状态，并将 state 属性设置为该状态的名称，以指定 Item 的当前状态。

我们模拟一下交通信号灯，分别用 go、wait、stop 表示行驶、警示和停止状态，并通过鼠标点击来控制各状态之间的切换：

import QtQuick
import QtQuick.Controls

Rectangle {
            
    id: light
    width: 150
    height: width
    radius: width / 2
    state: "go"  // 初始状态

    // 交通信号灯（红黄绿）
    states: [
        State {
            
            name: "go"
            PropertyChanges {
             target: light; color: "green" }
        },
        State {
            
            name: "wait"
            PropertyChanges {
             target: light; color: "yellow" }
        },
        State {
            
            name: "stop"
            PropertyChanges {
             target: light; color: "red" }
        }
    ]

    // 状态开关（用于状态切换）
    MouseArea {
            
        anchors.fill: parent
        onClicked: {
            
            if (light.state === "go") {
            
                light.state = "wait"
            } else if (light.state === "wait") {
            
                light.state = "stop"
            } else {
            
                light.state = "go"
            }
        }
    }
}

在这个例子中，我们定义了一个圆形，用于表示信号灯。

因为存在三种状态：行驶、警示和停止，所以我们定义了对应的三个 State 对象：go、wait、stop，并通过 PropertyChanges 来更改 light 组件的 color 属性值。当状态为 go 时，背景色为绿色；当状态为 wait 时，背景色为黄色；而当状态为 stop 时，背景色为红色。

在定义完这几种状态之后，我们将 state 属性的初始值指定为 go，因此信号灯的初始背景色是绿色。

最后，我们使用了 MouseArea 类型作为状态开关。当鼠标点击时，信号灯的状态会在 go、wait、stop 之间不断切换，这样就实现了背景色变化的效果。

自动切换状态

为了便于我们使用，State 元素提供了一个 when 属性，用于指定状态何时被激活。它接受一个条件表达式，只有当条件为 true 时，相应的状态才会被激活；如果条件为 false，则状态会恢复到默认状态。

下面，我们创建一个蓝色矩形，并在鼠标按下时让其自动隐藏：

import QtQuick
import QtQuick.Controls

Rectangle {
            
    id: root
    width: 150
    height: 150
    color: "blue"

    states: State {
            
        name: "hidden"
        when: mouseArea.pressed
        PropertyChanges {
             target: root; opacity: 0 }
    }

    MouseArea {
            
        id: mouseArea
        anchors.fill: parent
    }
}

在这个例子中，矩形的默认状态是可见的，因为 opacity 属性的默认值是 1.0。

但当鼠标被按下时，when 所在表达式 mouseArea.pressed 就会为 true，这时矩形会自动切换至 hidden 状态。在该状态下，矩形的 opacity 属性值为 0，所以会隐藏起来。倘若随后释放鼠标，则表达式的值会变为 false，此时矩形会重新回到默认的可见状态。

扩展状态

有些时候，多个状态中可能存在一些重复的配置，这时我们可以使用 State 的 extend 属性将一个状态扩展到另一个状态。

简单理解，扩展就是在状态之间创建一种继承关系，其中一个状态可以继承另一个状态的属性和行为，被扩展的状态就是基状态（类似于 C++ 中的类继承）。

我们修改一下信号灯的例子，将 go、wait 状态下的不透明度设置为 0.3：

import QtQuick
import QtQuick.Controls

Rectangle {
            
    id: light
    width: 150
    height: width
    radius: width / 2
    state: "go"  // 初始状态

    // 交通信号灯（红黄绿）
    states: [
        State {
            
            name: "base"
            PropertyChanges {
             target: light; opacity: 0.3 }
        },
        State {
            
            name: "go"
            extend: "base"
            PropertyChanges {
             target: light; color: "green" }
        },
        State {
            
            name: "wait"
            extend: "base"
            PropertyChanges {
             target: light; color: "yellow" }
        },
        State {
            
            name: "stop"
            PropertyChanges {
             target: light; color: "red" }
        }
    ]

    // 状态开关（用于状态切换）
    MouseArea {
            
        anchors.fill: parent
        onClicked: {
            
            if (light.state === "go") {
            
                light.state = "wait"
            } else if (light.state === "wait") {
            
                light.state = "stop"
            } else {
            
                light.state = "go"
            }
        }
    }
}

这里，我们额外添加了一个状态 base，使其作为基状态，它定义了 light 组件的 opacity 属性值为 0.3。由于状态 go 和 wait 扩展自 base，因此它们继承了 base 的所有更改。当切换至 go 和 wait 状态时，信号灯的不透明度将是 0.3。

需要注意色是，stop 状态并没有扩展 base，所以当切换到 stop 时，信号灯的不透明度依然是 1.0。

状态改变

在使用 State 管理状态时，我们不仅可以改变元素的属性值，还可以进行其他类型的变化，其中有四种主要的状态改变类型：

  • PropertyChanges：指对象属性的改变，比如颜色或大小。
  • ParentChange：指对象父级关系的改变，就像把一个东西从一个盒子移到另一个盒子里一样。
  • AnchorChanges：指对象之间相对位置的改变，类似于调整物体之间的连接或定位。
  • StateChangeScript：可以执行一些脚本或代码，比如触发特定的行为或动作。

这些类型将被应用到 State 的 changes 属性上，该属性接收一个变化列表 list<Change>，其中保存了应用于该状态的所有变化。

由于 PropertyChanges 上面已经介绍过了，接下来我们看看其他几种类型的用法。

更改父对象

ParentChange 用于在状态切换时改变元素的父对象，它的核心属性是 target 和 parent，其中 target 是要更改的目标元素，parent 是该元素新的父对象。除了这两个属性之外，还可以通过 x、y、width、height 属性指定目标元素的新位置以及大小，甚至是通过 scale 和 rotation 属性进行缩放和旋转。

下面的示例，我们通过鼠标点击来更改矩形的父对象：

import QtQuick
import QtQuick.Controls

Rectangle {
            
    width: 200; height: 150
    color: "lightgray"

    Rectangle {
            
        id: greenRect
        width: 125; height: 150
        color: "green"
    }

    Rectangle {
            
        id: blueRect
        x: greenRect.width
        width: 75; height: 75
        color: "blue"

        states: State {
            
            name: "reparented"
            ParentChange {
            
                target: blueRect
                parent: greenRect
                x: 10; y: 10
            }
        }

        MouseArea {
            
            anchors.fill: parent
            onClicked: blueRect.state = "reparented"
        }
    }
}

这里一共有三个矩形，最外层的是一个灰色矩形，里面包含了一个较大的绿色矩形和一个较小的蓝色矩形，它们水平放置在一起。也就是说，灰色矩形是它们的父对象。

在蓝色矩形内部，我们定义了一个名为 reparented 的状态。通过使用 ParentChange，我们为 blueRect 重新指定了一个父对象 greenRect，并将其位置设置在坐标点 (10, 10) 处。

当单击蓝色矩形时，它会切换到 reparented 状态，这样就完成了最终的父对象变更的效果。

更改锚点

AnchorChanges 用于在状态切换时改变元素的相对位置，它有两个属性：target 和 anchors，其中 target 是要更改的目标元素，而 anchors 是一个属性组，包含了 left、right、top、bottom 等子属性，这些属性定义了目标元素在不同状态下的位置锚点。

下面的例子，我们通过鼠标点击来更改矩形的的锚点：

import QtQuick
import QtQuick.Controls

Rectangle {
            
    width: 150
    height: 150
    color: "lightgray"

    Rectangle {
            
        id: blueRect
        width: 75
        height: 75
        anchors.left: parent.left
        anchors.top: parent.top
        color: "blue"

        states: State {
            
            name: "reanchored"

            AnchorChanges {
            
                target: blueRect
                anchors.left: undefined
                anchors.top: undefined
                anchors.right: parent.right
                anchors.bottom: parent.bottom
            }
        }

        MouseArea {
            
            anchors.fill: parent
            onClicked: blueRect.state = "reanchored"
        }
    }
}

这里我们定义了一个灰色矩形和一个蓝色矩形，蓝色矩形通过 anchors.left: parent.left 和 anchors.top: parent.top 锚定在灰色矩形的左上角。

在蓝色矩形内部，我们定义了一个名为 reanchored 的状态。通过使用 AnchorChanges，我们将其从原来的位置移动到了父窗口的右下角（anchors.right: parent.right 和 anchors.bottom: parent.bottom）。为了保持其大小不变，我们还同时使用了 undefined 来重置左上角（anchors.left 和 anchors.top）的锚点。

当点击蓝色矩形时，它会切换到 reanchored 状态，这样就完成了最终的位置变化。

执行脚本

StateChangeScript 用于在状态切换时执行一些脚本或代码，它有两个属性：name 和 script，其中 name 是脚本的名称（动画 ScriptAction 会用到），而 script 是具体要执行的脚本。

下面的例子，我们通过鼠标点击来运行一些脚本：

import QtQuick
import QtQuick.Controls

Rectangle {
            
    id: root
    width: 150
    height: 150
    color: "blue"

    states: State {
            
        name: "runscript"
        StateChangeScript {
            
            script: {
            
                console.info("state changed")
                root.grabToImage(function(result) {
            
                    result.saveToFile("image.png");
                });
            }
        }
    }

    MouseArea {
            
        anchors.fill: parent
        onClicked: root.state = "runscript"
    }
}

这里，我们定义了一个名为 runscript 的状态。通过使用 StateChangeScript，我们执行了一些脚本，该脚本的作用是打印一条简单的调试信息，并抓取窗口图片然后保存为 image.png 文件。

当点击蓝色矩形时，它会切换到 runscript 状态，这样就完成了脚本的执行。
