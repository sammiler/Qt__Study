本文介绍了QML中Image元素的基本用法，包括显示本地和网络图片、控制图像大小、加载尺寸、图像反转、填充模式等。通过示例代码展示了如何通过sourceSize属性调整图像加载尺寸，以及如何使用fillMode属性控制图像的显示方式。此外，还提到了如何加载网络图片，并对加载状态进行监控，以及在网络图片加载时的性能优化考虑。

在 QML 中，Image 是用于显示图像的基本元素之一，通过它可以很容易地加载本地或者网络图片。Image 支持多种标准图像格式，包括位图（如 PNG 和 JPEG）和矢量图（如 SVG）。如果需要显示动图，可使用 AnimatedSprite 或者 AnimatedImage。

以下是一些常用的 Image 属性：

属性 描述
source 用于指定显示的图像路径，可以是本地文件的路径或远程 Url 地址。
mirror 用于控制图像是否镜像显示（即：水平反转）
mirrorVertically 用于控制图像是否镜像显示（即：垂直反转）
fillMode 用于控制图像的显示方式（例如，使用 Image.PreserveAspectFit 来保持图像的宽高比，并缩放图像以适合可视区域）
asynchronous 用于指定图像加载时是否启用异步模式。若为 true，则异步加载，这意味着在后台进行且不会阻塞主线程，从而提高程序的性能和响应能力；若为 false，则同步加载，直到图像加载完成，这可能导致程序无响应。
status 表示当前图像的加载状态（例如：在加载图像出错时状态为 Image.Error）
progress 表示当前图像的加载进度

基本用法

先看一个简单的示例，在 QML 界面中显示一张名为 logo.jpg 的图像：

import QtQuick

Image {
            
    source: "qrc:/images/logo.jpg"
}

在上述代码中，我们创建了一个 Image 元素，并为 source 属性指定了一个以 qrc 为前缀的图片路径，这说明我们引用的是 qrc 中的资源文件。由于我们没有指定大小，因此图像在显示时将自动使用其原始大小。

控制显示图像的大小

如果需要控制显示图像的大小，可以使用 width 和 height 属性：

Image {
            
    width: 100
    height: 100
    source: "qrc:/images/images/logo.jpg"
}

这里，我们将 width 和 height 属性均设置为 100，这将使图像的宽度和高度为 100 像素。如果仅设置其中的一个属性，则另一个将根据图像的宽高比自动调整。

控制加载图像的尺寸

sourceSize 属性用于指定实际加载的图像的尺寸，它改变的是加载的图像的大小：

Image {
            
    source: "qrc:/images/images/logo.jpg"
    sourceSize: Qt.size(100, 100)
}

虽然这里设置的大小也是 100 x 100，但由于我们是将一个小的 sourceSize 应用于一个较大的图像，因此图像看起来比较模糊。

注意： 当图像的分辨率较大时，使用该属性可以缩小图像的大小，以减少内存占用和加载时间，但可能会影响图像质量。

图像反转

有时，我们可能需要有效的显示镜像，那么可以利用 mirror 和 mirrorVertically 属性对图像进行水平反转和垂直反转：

Row {
            
    spacing: 10

    // 水平反转
    Image {
            
        source: "qrc:/images/logo.jpg"
        mirror: true
    }

    // 垂直反转
    Image {
            
        source: "qrc:/images/logo.jpg"
        mirrorVertically: true
    }
}

填充模式

当源图像的大小与项目的大小不同时，可以通过 fillMode 属性控制图像的显示方式，来达到缩放和对齐图片以适应 Image 元素的内容矩形（即可视区域）的目的。

该属性包括以下几种选项：

选项 描述
Image.Stretch（默认值） 缩放图像以填充可视区域，无需保持宽高比。
Image.PreserveAspectFit 保持宽高比，并缩放图像以适合可视区域。
Image.PreserveAspectCrop 保持图像宽高比，并缩放图像以填充可视区域（必要时进行裁剪）。
Image.Tile 将原始图像平铺在可视区域中（即：当图像大小小于可视区域时，图像会水平和垂直复制）。
Image.TileVertically 水平居中重复显示元素，直到填充整个可视区域（与“Tile”不同，此选项会垂直平铺，但在水平方向上只显示一次）。
Image.TileHorizontally 垂直居中重复显示元素，直到填充整个可视区域（与“Tile”不同，此选项会水平平铺，但在垂直方向上只显示一次）。
Image.Pad 图像未转换

以下是一个示例，演示了不同的 fillMode 选项：

Row {
            
    spacing: 10
    Image {
            
        width: 120; height: 80
        source: "qrc:/images/logo.jpg"
    }
    Image {
            
        width: 120; height: 80
        source: "qrc:/images/logo.jpg"
        fillMode: Image.PreserveAspectFit
    }
    Image {
            
        width: 120; height: 80
        source: "qrc:/images/logo.jpg"
        fillMode: Image.PreserveAspectCrop
    }
    Image {
            
        width: 150; height: 150
        source: "qrc:/images/logo.jpg"
        fillMode: Image.Tile
        horizontalAlignment: Image.AlignLeft
        verticalAlignment: Image.AlignTop
    }
    Image {
            
        width: 150; height: 150
        source: "qrc:/images/logo.jpg"
        fillMode: Image.TileVertically
        verticalAlignment: Image.AlignTop
    }
    Image {
            
        width: 150; height: 150
        source: "qrc:/images/logo.jpg"
        fillMode: Image.TileHorizontally
        horizontalAlignment: Image.AlignLeft
    }
}

加载网络图片

除了访问本地图片文件之外，使用 Image 元素还可以轻松地显示网络图片，只需要在 source 属性中指定远程 Url 地址即可。

下面，我们创建一个灰色矩形，并通过鼠标点击来实现网络图片的加载：

Rectangle {
            
    width: 100
    height: 100
    color: "gray"

    Image {
            
        id: image
        anchors.fill: parent
        // 是否缓存
        cache: false
        // 网络加载总是异步的，无需指定
        // asynchronous: true

        // 显示加载状态
        onStatusChanged: {
            
            if (image.status === Image.Loading) {
            
                console.info("loding....");
            } else if (image.status === Image.Ready) {
            
                console.info("ready....");
            } else if (image.status === Image.Error) {
            
                console.info("error....");
            } else {
            
                console.info("status:" + image.status);
            }
        }
    }

    // 点击鼠标，加载图片
    MouseArea {
            
        anchors.fill: parent
        onClicked: {
            
            image.source = "https://profile-avatar.csdnimg.cn/5fff51893a944afa996a7393b09c2a78_u011012932.jpg"
        }
    }
}

在这个示例中，我们为矩形中添加了一个 MouseArea，一旦鼠标被点击，就会为 Image 的 source 属性绑定远程 URL，这时网络图片会开始加载。为了监控加载的状态，我们特意添加了相应的 onStatusChanged 处理程序，并将结果打印到了控制台上。

一般情况下，在加载网络图片时，可能会出现网络问题或文件无法加载，因此可根据具体情况自定义错误处理程序。另外，在加载大量网络图片时，可能会出现性能问题，建议对图片进行缓存或懒加载，以优化用户体验。
