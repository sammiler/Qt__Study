本文详述了Qt6 for Android环境搭建的全过程，包括核心概念、安装JDK、Android SDK和NDK，以及在Qt Creator中配置Android工具和创建AVD。通过配置Android SDK、NDK与Qt Creator的集成，解决环境搭建中的常见问题，确保Qt应用能顺利运行在Android设备上。

说起 Qt 各平台的开发环境搭建，最难的当属 Android 环境，因为要配置的东西比较多，而且配置过程很容易出现问题。例如，JDK、Android SDK、NDK 等必选工具，一旦选择的版本不匹配，就会提示各种错误。此外，还有像 OpenSSL 等一些可选工具，也需要进行配置，否则也会提示错误。

针对 Android 环境，虽然 Qt 官网提供了相关资料（例如：Qt for Android ），但只是一些简单的说明，而实际配置过程要复杂很多，需要趟不少的坑才行。

为了帮助大家顺利地完成 Qt6 for Android 的环境搭建，下面我会尽可能地讲解所涉及的每一个核心概念，并详细地列出安装、配置过程中的基本步骤，以及常见的一些注意事项和问题解决方案。

核心概念

随着 Qt6.5 的发布，最大的特性之一就是增加了对 Android 12 的支持，具体支持的配置如下：

分发 架构 编译器
Android 8.0 ~ Android 13 (API level 26 ~ API 33) armeabi-v7a, arm64-v8, x86 和 x86_64 Clang 14.0.6 (NDK r25b)

在稍后 Qt6 for Android 的环境搭建中，会涉及一些重要的工具，我们做一下简单的介绍：

术语 描述
JDK Java Development Kit，用于支持 Android 开发环境的运行
Android Studio 用于安装和管理 Android SDK 和 NDK，提供开发工具和模拟器。
Android SDK Android 软件开发工具包，包含 Android 平台相关的库和工具。
Android NDK Android Native Development Kit，用于支持使用 C/C++ 进行 Android 开发。

Android SDK 配置文件

在开始之前，必须得介绍一下 sdk_definitions.json（所在路径：%QTDIR%\Tools\QtCreator\share\qtcreator\android ），它是 Qt Creator 中用于定义 Android SDK 的配置文件。

这个文件包含了 Android SDK 的路径、版本号、构建工具版本等信息，目的是告诉 Qt Creator 在构建和部署 Android 项目时应该使用哪个 Android SDK 版本和相关工具，内容如下：

也就是说，我们要下载的工具以及它们的版本号，最好和文件中的保持一致，否则就有可能出错。

安装 JDK

  1. 下载 JDK

现在要正式开始了，首先需要安装 JDK（或者 OpenJDK）。进入 Java Downloads 页面 ，找到“Java 11” 进行下载安装：

注意： 下载时需要登录 Oracle 账户，所以没有的话需要创建一个。

  1. 配置环境变量

在安装完成之后，建议配置一下环境变量。右键计算机，选择【属性】->【高级系统设置】->【环境变量】->【系统变量】，然后进行新建：

变量 值
JAVA_HOME D:\DevelopTools\Java\jdk-11
Path %JAVA_HOME%\bin

安装 Android SDK 和 NDK

要获取 Qt for Android 的开发环境，可以选择以下两种方式：

  • 独立下载 Android SDK 和 NDK：独立地管理 Android 开发工具，但需要手动配置环境变量和 Qt Creator。
  • 通过 Android Studio 管理 Android SDK 和 NDK（推荐）：更加集成化，Qt Creator 会与 Android Studio 协同工作，减少了一些手动配置的过程。
  1. 安装 Android Studio

进入 Android Studio downloads 页面，下载安装 Android Studio：

  1. SDK 管理

打开 Android Studio，在“More Actions”下拉列表中选择“SDK Manager”：

  1. 配置 SDK Platforms

在配置界面中，需要先为“Android SDK Location”指定一个位置，随后就可以按照 sdk_definitions.json 中的内容选择具体的组件和对应的版本了。

点击“SDK Platforms”，选择“Android 12.0(“S”)”，对应的 API Level 是 31：

  1. 配置 SDK Tools

切换到“SDK Tools”页面，点击右下角的“Show Package Details”，这时会列出各组件的详细版本。

在“Android SDK Build-Tools 34”中，选择“31.0.0”版本：

在“NDK (Side by side)”中，选择“25.1.8937393”版本：

然后，再勾选一下“Android SDK Command-line Tools (latest)”等其他组件，就可以进行安装了：

当 JDK、Android SDK 和 NDK 全部安装完成之后，接下来要做的就是在 Qt Creator 中配置这些工具。

在 Qt Creator 中配置 Android 工具

  1. 配置 Android 工具

打开 Qt Creator，选择菜单栏中的【编辑】-> 【Preferences …】进入设置界面。

点击【设备】-> 【Android】，然后配置“JDK 位置”、“Android SDK 的路径”、“安卓 NDK 列表”：

如果提示“安卓设置存在错误”，不要着急，可能是 Command-line Tools (latest) 版本过高导致的问题。

  1. 替换 Command-line Tools

进入 Command-line Tools 安装目录（位置：%SDK%\cmdline-tools\latest ），删除其中的所有文件，然后按照 sdk_definitions.json 中的要求下载 https://dl.google.com/android/repository/commandlinetools-win-9123335_latest.zip 进行替换。

在替换完成之后，重启 Qt Creator，再次进入 Android 配置界面，这时会提示我们进行更新，点击“确定”即可：

  1. 下载 OpenSSL

当上面的包更新完成之后，安卓环境就基本设置完成了。

但此时还有个小问题 - OpenSSL 报错，虽然这是一个可选项，但是看着总让人不舒服。点击“下载 OpenSSL”按钮进行修复，安装完成之后错误就消失了：

  1. 创建 Android 构建套件

由于上面默认勾选了“为安卓工具链自动创建套件”，所以只要安卓配置成功，对应的 Android Kit 就会自动生成。进入“构建套件(Kit)”界面查看一下：

这里生成了四个 Android Kits，分别代表了不同的 Android 模拟器或设备架构，它们之间的区别在于目标设备的 CPU 架构：

  • android clang arm64-v8a：适用于 arm64-v8a 架构的设备，这是 64 位的 ARM 架构。
  • android clang armeabi-v7a：适用于 armeabi-v7a 架构的设备，这是 32 位的 ARM 架构。
  • android clang x86：适用于 x86 架构的模拟器或设备
  • android clang x86_64：适用于 x86_64 架构的模拟器或设备

选择适合目标设备的 Kit，程序才能够正常地在对应的设备上运行和调试。所以需要根据测试环境或目标设备的 CPU 架构，选择对应的 Kit 进行设置和构建。

另外，细心的小伙伴可能已经发现了，这四个 Android Kit 前面都有一个警告。这是因为我们还没有设置“Run Device”，一旦 AVD 配置完成（全称：Android Virtual Device，即：Android 运行的虚拟设备），警告就会随之消失。

在 Android Studio 中创建 AVD

  1. 创建虚拟设备

打开 Android Studio，点击“Virtual Device Manager”按钮，进入设备管理界面：

点击“Create Device”按钮，准备创建虚拟设备：

  1. 选择设备

选择一个设备类型，例如：Pixel 4：

  1. 选择系统镜像

选择一个系统镜像，例如：Android 12.0（API Level 为 31），进行下载：

  1. 配置虚拟设备

修改虚拟设备的名称和屏幕方向等配置，点击“Finish”按钮完成创建：

  1. 列出虚拟设备

随后就能看到创建好的虚拟设备了，可以点击“Start”按钮启动测试：

在 Qt Creator 中创建 AVD

  1. 添加 AVD

打开 Qt Creator 的设置界面，选择“设备”选项卡，可以看到，在 Android Studio 中创建的“Pixel 4” AVD 已经被自动识别出来了，是不是很智能，也很方便。

此外，我们也可以在 Qt Creator 中创建 AVD。点击右侧的“添加”按钮，在弹出的对话框红填写 AVD 的名称和路径等信息，然后保存配置就可以了：

完成之后，就能同时看到已创建的两个 AVD 设备信息了：

  1. Android Kits 配置

再次查看 Android Kits，可以看到，前面的警告已经消失了。这是因为在 AVD 添加完成之后，Qt Creator 会自动将其添加到“Run Device”中。

至此，我们的环境就完全配置好了。

选择任意一个 AVD 虚拟设备，例如：Glalaxy Nexus，然后运行程序：
