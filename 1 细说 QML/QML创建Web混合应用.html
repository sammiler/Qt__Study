本文介绍了如何利用QML的WebEngineView组件在原生应用中创建Web混合应用，展示了初始化Web引擎、加载页面及处理加载事件的步骤，并强调了Windows平台使用MSVC编译器的必要性。

作者： 一去、二三里
个人微信号： iwaleon
微信公众号： 高效程序员

随着互联网的快速发展，Web 应用在各个领域中变得越来越流行。为了满足用户对多样化功能的需求，我们经常需要将 Web 技术和原生应用相结合，来创建混合应用程序。

混合应用程序：是一种应用程序开发方法，它利用 Web 技术（HTML、CSS 和 JavaScript）来构建用户界面，并由包装器将 Web 界面嵌入到原生应用程序中，以便能够在不同平台和设备上运行。

而 QML 中的 WebEngineView 就是这样一个包装器，但它本身又是一种原生控件，用于在 QML 原生应用中显示 Web 内容并与之交互。

想象一下，打开一个应用程序，看到一个类似于网页的界面，并且可以像在浏览器中一样进行交互。但这个应用程序无需额外地安装浏览器，因为它已经内置了一个能够显示网页的组件，这就是 WebEngineView。

这种混合应用程序的优势在于，它既具备了 Web 技术的跨平台特性和开发效率，又能够利用原生应用程序提供的功能和性能优势，从而可以为用户提供更好的体验。

使用 WebEngineView

初始化 Web 引擎

要渲染 WebEngineView，必须在程序中使用 QtWebEngineQuick::initialize() 初始化 Web 引擎：

#include <QGuiApplication>
#include <QtWebEngineQuick>
#include <QQmlApplicationEngine>

int main(int argc, char *argv[])
{
            
    QCoreApplication::setAttribute(Qt::AA_ShareOpenGLContexts);

    // 初始化 Web 引擎
    QtWebEngineQuick::initialize();

    QGuiApplication app(argc, argv);

    QQmlApplicationEngine engine;
    engine.load(QUrl("qrc:/main.qml"));
    return app.exec();
}

加载 Web 页面

然后，就可以在 QML 中导入 QtWebEngine 模块并创建 WebEngineView 组件了。

为了加载页面，通常可以采用两种方式 - url 属性或者 loadHtml() 方法。以 url 属性为例，我们来加载一个博客地址：

import QtQuick
import QtQuick.Window
import QtWebEngine

Window {
            
    width: 1000
    height: 800
    visible: true

    WebEngineView {
            
        id: engineView
        anchors.fill: parent
        url: "https://waleon.blog.csdn.net/"
    }
}

这个地址是一个网络 URL，它的前缀是“https://”；倘若加载的是本地网页，则应将前缀改为“file://”；类似的，如果是资源文件，则应使用“qrc://”作为前缀。

处理加载事件

在 WebEngineView 加载页面的过程中，可以根据一些属性、信号来处理具体的加载事件。例如，可以通过 loadProgress 属性来获取页面的加载进度（范围：0 ~ 100），也可以通过 loadingChanged(WebEngineLoadingInfo loadingInfo) 信号监听页面的加载状态。

注意信号中的参数 WebEngineLoadingInfo，它是一个对象类型，其中包含的属性有：

属性 描述
url 加载请求的 URL
status 加载请求的状态（枚举）
WebEngineView.LoadStartedStatus：页面正在加载
WebEngineView.LoadStoppedStatus：页面停止加载，调用了 stop() 方法，或者由 Chromium 中的加载器代码或网络堆栈触发
WebEngineView.LoadSucceededStatus：页面加载成功
WebEngineView.LoadFailedStatus：页面无法加载
errorCode 加载请求失败时（例如：URL 输入错误、断网）的错误码（整数）
errorString 加载请求失败时的错误信息（字符串）
errorDomain 加载请求失败时的错误类型（枚举）
QWebEngineLoadingInfo::NoErrorDomain：未知错误
QWebEngineLoadingInfo::InternalErrorDomain：内容不能被 Qt WebEngine 解析
QWebEngineLoadingInfo::ConnectionErrorDomain：网络连接故障导致的错误
QWebEngineLoadingInfo::CertificateErrorDomain：SSL/TLS 证书错误
QWebEngineLoadingInfo::HttpErrorDomain：HTTP 连接错误
QWebEngineLoadingInfo::FtpErrorDomain：FTP 连接错误
QWebEngineLoadingInfo::DnsErrorDomain：DNS 连接错误
QWebEngineLoadingInfo::HttpStatusCodeDomain：错误是 HTTP 响应状态码，即使在成功的情况下，例如：服务器以状态 200 响应

现在，我们添加一些简单的逻辑，在页面加载时打印具体的状态和进度：

WebEngineView {
            
    id: engineView
    anchors.fill: parent
    url: "https://waleon.blog.csdn.net/"

    // 加载状态改变（开始、停止、成功、失败）
    onLoadingChanged: (loadingInfo)=> {
            
        switch (loadingInfo.status) {
            
            case WebEngineView.LoadStartedStatus:
                console.info("load started: " + loadingInfo.url);
                break;
            case WebEngineView.LoadStoppedStatus:
                console.info("load stopped: " + loadingInfo.url);
                 break;
            case WebEngineView.LoadSucceededStatus:
                console.info("load succeeded: " + loadingInfo.url);
                break;
            case WebEngineView.LoadFailedStatus:
                console.error("load failed, errorCode:%1, errorDomain:%2, errorString:%3"
                              .arg(loadingInfo.errorCode).arg(loadingInfo.errorDomain).arg(loadingInfo.errorString));
                break;
            default:
                break;
            }
    }

    // 加载进度改变（0 ~ 100）
    onLoadProgressChanged: {
            
        console.info("load progress: " + loadProgress + "%");
    }
}

注意事项

如果是 Windows 平台，在使用 WebEngineView 时，要选择 MSVC 编译器。倘若使用 MinGW，会提示以下错误：

error: Unknown module(s) in QT: webenginequick

至于原因，官方文档中有一个重点说明：

The Qt WebEngine core is based on the Chromium Project.

由于 WebEngine 基于 Chromium 引擎（由谷歌开发的开源浏览器引擎，负责解析和渲染网页内容），而 Chromium 项目（由谷歌主导的开源项目，旨在开发和维护 Chromium 引擎）在 Windows 上使用了 MSVC 进行构建。为了保持二进制兼容性和稳定性，并与底层的 Chromium 引擎保持一致，Qt 官方同样推荐使用 MSVC 来构建 WebEngine。这样可以确保插件与底层引擎相匹配，避免潜在的兼容性问题。

因此，Qt 并未提供针对 MinGW 版本的 WebEngine 相关模块。但若真想在 MinGW 环境中使用 WebEngine，也并非绝对不可行，只不过需要手动构建和集成 Chromium 引擎，并将其与 Qt 和 MinGW 进行适配。但这个过程相对复杂，需要进行大量的配置和调整，因此并不是官方默认支持的选项。
