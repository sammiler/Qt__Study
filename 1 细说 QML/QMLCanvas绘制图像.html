本文详细介绍了如何在QML的Canvas上绘制图像，包括在指定点、指定位置和区域以及截取部分图像进行绘制。同时，文章还提到了绘制时如何避免图像拉伸变形的问题，以及图像资源可以是URL、Image元素或CanvasImageData对象。

对于 QML Canvas 而言，除了前面讲解的点线面的基本绘制之外，它还提供了一项很有意思的特性 - 绘制图像。只要是 Qt 支持的图片格式都可以使用，比如 PNG、JPEG 等，甚至可以将其它 Canvas 元素生成的图片作为图像资源。

关于图像的绘制，Context2D 提供了三种方法：

方法 描述
drawImage(variant image, real dx, real dy) 将给定的图像绘制到画布的 (dx, dy) 位置处
drawImage(variant image, real dx, real dy, real dw, real dh) 将给定的图像绘制到画布上，位置为 (dx, dy)，宽度为 dw，高度为 dh
drawImage(variant image, real sx, real sy, real sw, real sh, real dx, real dy, real dw, real dh) 将给定的 image 从 (sx, sy) 位置处截取，截取的宽度为 sw，高度为 sh，并将其绘制到画布的 (dx, dy) 位置处，绘制的宽度为 dw，高度为 dh。

这里的 image 指的是绘制到画布上的图像资源，可以是 Image 元素或者 Canvas 元素，也可以是一个图片 URL 或者一个 CanvasImageData 对象。当作为 Image 元素时，如果图像没有完全加载，则不绘制任何内容。当作为 URL 字符串时，图像应该先通过调用 Canvas::loadImage() 方法来加载。

完整的思维导图如下：

在画布的指定点绘制图片

我们先来演示第一个图像绘制方法，从画布的坐标点 (100, 100) 处开始绘制一张图片：

import QtQuick

Canvas {
            
    width: 200; height: 200

    property string img: "qrc:/images/logo.jpg"

    onPaint: {
            
        var ctx = getContext("2d")

        // 绘制图片
        ctx.drawImage(img, 100, 100)
    }

    // 图片加载完成时，重绘 Canvas
    onImageLoaded: requestPaint()

    // 在附加处理器中加载图片
    Component.onCompleted: loadImage(img)
}

在这个例子中，图像资源是一个图片 URL（本地或者远程地址均可）。由于图片在绘制之前需要先被加载，因此我们在附加处理器中调用了 Canvas::loadImage()，同时监听了 imageLoaded() 信号并在对应的处理程序中调用了 requestPaint()。这样一来，当图片加载完成时，就会对整个可见区域请求重绘。

和之前一样，真正的绘制是在 onPaint 事件中进行的，我们通过调用 drawImage() 从画布的中心点 (100, 100) 处开始绘制图像，但由于画布的实际绘制区域（100 x 100）比图像本身小（128 x 128），所以导致图片显示不全，看起来像是被裁剪了一样。

在画布的指定位置和区域绘制图片

由于上述方式在使用上不够灵活，因此就有了第二个图像绘制方法，通过它不但可以指定绘图位置，还能够指定绘制区域。

例如，从画布的坐标点 (50, 50) 处开始，在 80 x 120 区域内绘制一张图片：

Canvas {
            
    width: 200; height: 200

    property string img: "qrc:/images/logo.jpg"

    onPaint: {
            
        var ctx = getContext("2d")

        // 绘制图片（起点坐标、绘制区域）
        ctx.drawImage(img, 50, 50, 80, 120)
    }

    onImageLoaded: requestPaint()
    Component.onCompleted: loadImage(img)
}

看到没有，此时的图片被完整显示出来了，并且会自己缩放到了指定大小。由于这里指定的区域比例和原图不一样，所以图片被拉伸了，直到填满整个区域。

截取图片的一部分绘制到画布上

如果不想绘制整张图片，而是想在画布的指定区域显示图片的一部分内容，则可以使用第三个图像绘制方法。

下面的代码，我们截取了距离图片左上角位置 (20, 20) 宽高为 80 x 80 的部分图片，并将其绘制到画布上，绘制点为 (50, 50)，绘制宽高为 80 x 120：

Canvas {
            
    width: 200; height: 200

    property string img: "qrc:/images/logo.jpg"

    onPaint: {
            
        var ctx = getContext("2d")

        // 绘制图片（图像截取区域、画布绘制区域）
        ctx.drawImage(img, 20, 20, 80, 80, 50, 50, 80, 120)
    }

    onImageLoaded: requestPaint()
    Component.onCompleted: loadImage(img)
}

可以看到，截取的部分图片被绘制出来了，但由于绘制的宽高 dw/dh 与图像截取的宽高 sw/sh 不一致，导致图像被缩放变形了。

如果要保持两者大小一致（即：dw = sw，dh = sh），最简单的办法是直接省略绘制的宽高 dw/dh：

drawImage(img, 20, 20, 80, 80, 50, 50)

这段代码等价于：

var sw = 80
var sh = 80
var dw = sw
var dh = sh

ctx.drawImage(img, 20, 20, sw, sh, 50, 50, dw, dh)

通过以上总结，建议在 Canvas 上绘制图片时，应最好确保绘制区域和图片大小相适应，否则图片会被拉伸变形。

其他绘图方式

在 Canvas 绘制图像时，图像资源除了使用 URL 字符串之外，还可以使用 Image 元素，或者 CanvasImageData 对象（涉及像素原理，后面分享）。

以下是一个基于 Image 元素的示例：

Canvas {
            
    width: 200; height: 200

    Image {
            
        id: image
        source: "qrc:/images/logo.jpg"
        visible: false  // 先将其隐藏
    }

    onPaint: {
            
        var ctx = getContext("2d")

        // 绘制图片
        ctx.drawImage(image, 100, 100)
    }
}

这里，我们在 Canvas 中创建了一个辅助元素 Image，并将其 visible 属性设置为 false，这样它就不会显示在屏幕上。接下来，只需要将 drawImage() 中的第一个参数指定为这个 Image 即可，这样就完成了图像的绘制。

需要注意的是，如果 Image 元素加载的是网络图片，需要监听图片的加载状态，当 status 变为 Image.Ready 时，应调用 Canvas::requestPaint() 从而才能完成图片的绘制。
