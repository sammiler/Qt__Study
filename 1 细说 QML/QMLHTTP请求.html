本文介绍了在 QML 中使用 XMLHttpRequest 进行 HTTP 请求的详细步骤，包括异步和同步请求、请求状态跟踪、头信息处理以及响应数据的获取。通过示例展示了如何发送 GET、POST 请求，并提供了调试工具 httpbin 的介绍，帮助开发者理解和调试网络应用。

在 Web 开发中，实现与服务器的通信至关重要，其中 HTTP 便是最常用的方式之一。它是一种客户端 - 服务器协议，客户端向服务器发送请求，服务器则返回响应。常被用于在浏览器/客户端和 Web 服务器之间传输消息、图像、音视频和其他类型的文件。

为了实现 HTTP 通信，QML 提供了一个 XMLHttpRequest 类型。它最初是为 JavaScript 设计的，QML 在其基础上进行了封装，所以两者的作用和使用方式大致相同。尽管 XMLHttpRequest 的名称中有“XML”一词，但实际上它可以操作任何数据格式，如 JSON、XML 和文本等。

调试工具

要进行 HTTP 请求，先要有一个 HTTP Server。可以选择自己搭建，也可以使用已有的，这里推荐一个超级好用的工具 - httpbin。

这是一个基于 Python + Flask 的开源项目，目的是提供一个简单的 HTTP 请求和响应服务，以帮助我们测试和调试 HTTP 请求。

  • 官方网站：http://httpbin.org/
  • 开源地址：https://github.com/postmanlabs/httpbin

httpbin 的功能很强大，除了支持 HTTP 和 HTTPS 之外，还能模拟各种请求（例如：GET/POST/PUT/DELETE 等）、文件上传和下载、身份验证和授权流程等功能，同时会返回与请求相关的详细信息，包括：请求的 URL、方法、头部、主体、响应状态码和响应头等。

通过使用 httpbin，我们可以实时测试和验证 HTTP 请求，并了解其中的一些处理机制，这对于开发 API、调试网络应用和测试 HTTP 客户端非常有用。同时，httpbin 还提供了充分的文档和示例，来帮助开发者快速上手。

基本步骤

XMLHttpRequest 支持两种 HTTP 请求模式：同步和异步。

我们优先介绍一下异步请求，因为这是绝大多数情况下使用的模式，通常需要四个步骤：

  1. 创建一个 XMLHttpRequest 对象：
let xhr = new XMLHttpRequest()
  1. 为 onreadystatechange 事件分配一个回调函数，该函数会在请求状态（readyState）改变时被触发：
xhr.onreadystatechange = function() {
            
    if (xhr.readyState === XMLHttpRequest.DONE) {
            
        if (xhr.status === 200) {
            
            // 请求成功，处理 response、responseText 或者 responseXML
        } else {
            
            // 请求失败，处理 status、statusText
        }
    }
}

最好的做法是，在回调中跟踪 readyState 的变化，并在请求完成后（状态变为 DONE）根据 HTTP 状态码（例如：200、404 等）来判断请求是否成功，然后再进行相应的业务处理和错误提示。

  1. 调用 open() 方法，指定请求的参数：
xhr.open(method, url, async)
  • method：请求使用的 HTTP 方法，通常是 “GET” 或者 “POST”。
  • url：要请求的 URL
  • async：请求的模式，true 表示同步（默认值），false 表示异步。
  1. 调用 send() 方法，将请求发送给服务器：
xhr.send(data)

其中，data 是一个可选参数，用于指定请求的 body 体，这对于 POST 请求很有用。

异步请求

来看一个完整的示例，我们以异步方式发送一个 HTTP GET 请求，并将响应的内容输出到界面上：

import QtQuick
import QtQuick.Controls
import QtQuick.Layouts
import "qrc:/js/httprequest.js" as XHR

Item {
            
    width: 800
    height: 600

    ColumnLayout {
            
        anchors.fill: parent
        anchors.margins: 10
        spacing: 10

        RowLayout {
            
            anchors.margins: 10
            spacing: 10

            // 请求的 URL 输出框
            TextField {
            
                id: urlField
                placeholderText: qsTr("http:// or https://")
                text: "http://httpbin.org/get"
            }

            // 发送请求
            Button {
            
                Layout.alignment: Qt.AlignCenter
                text: "Send"
                onClicked: {
            
                    textArea.clear()
                    inner.sendRequestAsync(urlField.text)
                }
            }
        }

        // 显示 HTTP 响应
        TextArea {
            
            id: textArea
            Layout.fillWidth: true
            Layout.fillHeight: true
            placeholderText: qsTr("HTTP response output")
        }
    }

    QtObject {
            
        id: inner

        // 发送 HTTP 请求
        function sendRequestAsync(url) {
            
            XHR.sendRequestAsync(url, successfulCallback, failedCallback)
        }

        // 请求成功时的回调函数
        function successfulCallback(response) {
            
            textArea.append("========== headers ==========")
            textArea.append(response.headers)
            textArea.append("========== content ==========")
            textArea.append(response.content)
        }

        // 请求失败时的回调函数
        function failedCallback(error) {
            
            textArea.append("status:" + error.status)
            textArea.append("statusText:" + error.statusText)
        }
    }
}

这里，我们分别创建了一个输入框 urlField、按钮、和显示区域 textArea。其中 urlField 用于控制要请求的 URL，按钮用于发送 HTTP 请求，而 textArea 用于显示 HTTP 响应的具体信息。

当点击按钮时，会立即清空显示区域，并调用 sendRequestAsync() 函数。而该函数调用了一个外部的 XHR.sendRequestAsync() 函数，它接受 3 个参数，其中第一个是一个 url，第二个是一个请求成功时的回调函数，第三个是一个请求失败时的回调函数。关于这个函数，则是在 httprequest.js 中定义的，具体实现如下：

.pragma library

function sendRequestAsync(url, successfulCallback, failedCallback) {
            
    // 1. 创建 XMLHttpRequest 对象
    let xhr = new XMLHttpRequest()

    // 2. 为 onreadystatechange 事件分配一个回调函数
    xhr.onreadystatechange = function() {
            

        // 在请求完成后，根据 HTTP 状态码来判断请求是否成功，然后进行相应的业务处理和错误提示。
        if (xhr.readyState === XMLHttpRequest.DONE) {
            
            if (xhr.status === 200) {
            
                let response = {
            
                    headers : xhr.getAllResponseHeaders(),
                    content : xhr.response
                }

                successfulCallback(response)
            } else {
            
                let error = {
            
                    status : xhr.status,
                    statusText : xhr.statusText
                }

                failedCallback(error)
            }
        }
    }

    // 3. 指定 HTTP 请求参数
    xhr.open("GET", url)

    // 4. 将请求发送给服务器
    xhr.send()
}

这段代码遵循前面提到的四个步骤：先实例化 XMLHttpRequest 对象，然后为 onreadystatechange 事件分配一个回调函数来处理响应，接着指定 HTTP 请求参数，最后调用 open() 发送请求。

由于我们进行的是异步请求，所以这里采用了一种很常用的手法 - 回调函数。通过 successfulCallback 和 failedCallback，外部可以很容易地对请求成功和请求失败的结果做出相应的处理。

同步请求

如果想将异步请求改为同步，可以将 open() 方法中的 async 参数设置为 false。

由于是同步是阻塞式的，所以这时并不需要监听 onreadystatechange 事件，直接在 send() 之后处理响应的逻辑即可。

let xhr = new XMLHttpRequest();
xhr.open("GET", url, false);
xhr.send();

if (xhr.status === 200) {
            
    // 请求成功
} else {
            
    // 请求失败
}

相比异步请求，这在代码上看起来好像简洁了一些，但是缺点也非常明显：在服务器响应之前，GUI 会被阻塞，严重的话，可能还会导致程序挂起或者停止。正出于这个原因，一般几乎很少使用同步请求。

请求状态

随着 HTTP 请求的发送，请求的状态会不断地切换，可以通过 readyState 属性来访问当前的状态，取值如下：

枚举 描述
XMLHttpRequest.UNSENT 请求尚未初始化（调用 open() 之前的状态）
XMLHttpRequest.OPENED 请求已初始化（调用了 open()，但是还没调用 send()）
XMLHttpRequest.HEADERS_RECEIVED 已接收响应头信息（send() 被调用，响应头和响应状态已返回）
XMLHttpRequest.LOADING 下载响应中（响应体下载中，已经从 responseText 中获得部分数据）
XMLHttpRequest.DONE 请求完成（整个请求完成，无论成功或者失败，都算完成）

具体切换的顺序为：0 -> 1 -> 2 -> 3 -> … -> 3 -> 4，每当接收到网络数据包时，状态 3 都会重复。

如果要跟踪 readyState 的状态，可以在 readystatechange 事件中进行：

xhr.onreadystatechange = function() {
            
    // ......
    console.info("ready state:" + xhr.readyState)
}

头信息

HTTP headers，是 HTTP 传递附加信息的一种机制，它们包含了与请求或者响应相关的元数据，用于控制和描述消息的处理方式。例如，Accept 和 Content-Type，用于在客户端和服务器之间协商所接受或者发送的数据格式；User-Agent 和 Referer，用于告知服务器请求的来源和客户端的特性。

当这些标准头无法满足我们的要求时，可以添加自定义报头来进一步地扩展。例如，使用 X-Auth-Token 传递身份验证令牌，X-App-Version 标识客户端程序的版本号。

在使用 XMLHttpRequest 时，我们既可以发送自定义头信息，也可以从响应中读取头信息，涉及的方法有 3 个：

方法 描述
void setRequestHeader(name, value) 用给定的名称和值设置请求头
string getResponseHeader(headerName) 获取指定名称的响应头
string getAllResponseHeaders() 获取所有的响应头

注意： 对于标准头来说，一部分是由用户自己设置的（例如：Content-Type），但也有一些是由系统管理的（例如：Host），为了用户安全和请求的正确性，XMLHttpRequest 不允许更改这一部分。

我们来演示一下，看看如何设置 HTTP 请求头信息，并获取服务器返回的响应头信息：

let xhr = new XMLHttpRequest()

xhr.onreadystatechange = function() {
            
    if (xhr.readyState === XMLHttpRequest.HEADERS_RECEIVED) {
            
        // 获取所有的响应头
        let headers = xhr.getAllResponseHeaders()

        // 获取指定的响应头
        let contentTypeHeader = xhr.getResponseHeader("content-length")

        console.info("headers: " + headers);
        console.info("content-length: " + contentTypeHeader);
    }
}

xhr.open("GET", url)

// 使用自定义头信息
xhr.setRequestHeader("X-Auth-Token", "xxxxxxxxxxxxxxxx")
xhr.setRequestHeader("X-App-Version", "1.0")

// 使用标准头信息
xhr.setRequestHeader("Content-Type", "application/x-www-form-urlencoded; charset=utf-8");

xhr.send()

其中，由 getAllResponseHeaders() 获取的所有响应头信息会以单行字符串的形式返回：

date: Thu, 10 Aug 2023 08:06:47 GMT
content-type: application/json
content-length: 486
connection: keep-alive
server: gunicorn/19.9.0
access-control-allow-origin: *
access-control-allow-credentials: true

头与头之间的换行符是“\r\n”，name 和 value 之间的分隔符是冒号“:”，后面还跟了一个空格，这些都是固定的规范，所以很容易拆分和解析。

响应

使用 responseType 属性，可以指定响应内容的格式：

类型 描述
“text” 或 “”（空字符串） 响应内容为 UTF-16 编码的字符串
“arraybuffer” 响应内容为包含二进制数据的 ArrayBuffer
“json” 响应内容为 JSON 对象形式
“document” 响应内容为 Document 对象形式（即：XML 或者 HTML 文档），可使用 responseXML 属性进行读取

例如，可以像下面这样，在发送 HTTP 请求之前设置 responseType，并在响应完成时根据具体的类型来处理结果：

let xhr = new XMLHttpRequest()

xhr.onreadystatechange = function() {
            
    if (xhr.readyState === XMLHttpRequest.DONE) {
            
        if (xhr.status === 200) {
            
            // 根据响应类型来处理
            if (xhr.responseType === "text") {
            
                var text = xhr.responseText
                console.info("response text:", text);
            } else if (xhr.responseType === "json") {
            
                var json = JSON.parse(xhr.responseText)
                console.info("json url:", json.url)
            } else if (xhr.responseType === "document") {
            
                var xmlDoc = xhr.responseXML
                // 文档的版本号
                var xmlVersion = xmlDoc.xmlVersion
                // 文档的编码
                var xmlEncoding = xmlDoc.xmlEncoding
                // 根节点
                var rootNode = xmlDoc.documentElement
                // 节点的属性列表
                var attributes = rootNode.attributes
                // 节点的子节点
                var childNodes = rootNode.childNodes
                // ......

                console.info("xml version:" + xmlVersion)
                console.info("xml encoding:" + xmlEncoding)
            } else {
            
                console.info("response others:" + xhr.response)
            }
        } else {
            
            console.log("error:", xhr.status)
        }
    }
}

xhr.open("GET", url)

// 设置响应类型为 JSON
xhr.responseType = "json"

xhr.send()

注意： 获取响应内容的属性有三个，responseText 返回的是一个字符串，responseXML 返回的是一个 XML Document，而 response 会根据 responseType 返回对应的 String、ArrayBuffer 或者 Document。

POST 请求

要发送 POST请求，我们需要将 open() 方法中的 method 方法指定为“POST”，并构造一个表单数据，然后通过 send(data) 方法将数据提交到服务器。

在此过程中，我们可以通过请求头 Content-Type 来指定表单数据发送时的编码格式，通常有以下三种：

  • application/x-www-form-urlencoded：适用于普通表单提交，键值对（key1=value1&key2=value2）以 URL 编码形式传输。
  • application/json：适用于发送 JSON 格式的数据。
  • multipart/form-data：适用于文件上传和带有文件的表单提交，可以同时传输多个部分的数据。

以常见的用户登录为例，我们需要验证 username 和 password，这时可以将 Content-Type 设置为 application/x-www-form-urlencode，以普通的表单形式发送给后台：

// 设置表单内容（一个或多个字段）
var formData = "username=" + "waleon" + "&password=" + "123456"

// 发送请求
xhr.open("POST", "https://httpbin.org/post")
xhr.setRequestHeader("Content-Type", "application/x-www-form-urlencoded; charset=utf-8");
xhr.send(formData);

如果更喜欢 JSON 格式，也可以使用 JSON.stringify() 将对象转换为 JSON 字符串，然后进行发送。

但此时 Content-Type 应该设置为 application/json，绝大多数服务器端框架会通过它自动解码 JSON：

let account = {
            
    name: "waleon",
    password: "123456"
}

let json = JSON.stringify(account)

xhr.open("POST", "https://httpbin.org/post")
xhr.setRequestHeader("Content-Type", "application/json; charset=utf-8");
xhr.send(json)

终止请求

若觉得本次请求没必要继续，可以使用 abort() 方法进行终止：

xhr.abort()

当请求被取消之后，status 的值会变为 0。
